## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user free-text field → MarkdownService | Untrusted Markdown text crosses into HTML rendering; it must be parsed with raw-HTML disabled and sanitized against a strict allowlist before it can reach any browser (web profile) or email client (email profile). |
| NuGet registry → build | Third-party parser/sanitizer code (Markdig, HtmlSanitizer) enters the trusted build and becomes the app's entire XSS-defense surface. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-65-01 | Tampering / Elevation of Privilege | MarkdownService — raw `<script>`/event-handler HTML typed into a field | mitigate | `.DisableHtml()` on the Markdig pipeline prevents literal-HTML-tag recognition (block- and inline-level); verified by the `<script>` and `<img onerror>` XSS theory cases (assert no live `<script` and no `<[^>]*\son\w+=` event-handler attribute) |
| T-65-02 | Tampering / Elevation of Privilege | MarkdownService — `javascript:`-scheme via `[text](url)` link syntax | mitigate | HtmlSanitizer `AllowedSchemes = {http, https, mailto}` applied via `UriAttributes = {href, src}` strips the disallowed scheme; verified by the `[Click me](javascript:...)` XSS case (javascript-scheme regex on href/src is false) |
| T-65-03 | Tampering / Elevation of Privilege | MarkdownService — `javascript:` via native CommonMark angle-bracket autolink (`<javascript:...>`), NOT blocked by `.DisableHtml()` (confirmed via Markdig source) | mitigate | Same `AllowedSchemes`/`UriAttributes` mechanism — the ONLY defense for this vector, so it has a dedicated XSS theory case (`<javascript:alert(...)>`) distinct from the `[text](url)` case |
| T-65-04 | Tampering / Elevation of Privilege | MarkdownService — attribute injection via Markdig Generic Attributes syntax (`{onmouseover="..."}`) | mitigate | Never call `.UseGenericAttributes()` (and never `.UseAdvancedExtensions()`, which bundles it) — the `{...}` syntax renders inert as literal text; verified by the generic-attribute XSS case |
| T-65-05 | Tampering (DOM clobbering) | MarkdownService — attacker-controlled `id` attribute values | accept | `id` is allowlisted ONLY to make footnote jump-links work; Markdig emits only sequential `fn:N`/`fnref:N` values — there is no Markdown syntax (absent generic attributes) to set an arbitrary `id`, bounding the risk to non-attacker-controlled tokens |
| T-65-06 | Tampering (data integrity of legitimate user content) | HtmlSanitizer `KeepChildNodes = false` silently deleting whole subtrees when a container tag is missing from the allowlist | mitigate | Exhaustive per-extension `AllowedTags` (table family, dl/dt/dd, sup, div, input) verified against actual renderer output; per-extension content-survival unit tests assert the cell/definition/footnote TEXT survives sanitization, not merely that output is non-empty |
