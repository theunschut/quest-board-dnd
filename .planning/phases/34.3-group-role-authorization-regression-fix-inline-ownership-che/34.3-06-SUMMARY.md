## Decisions Made
- The plan's Task 2 note anticipated the `/hangfire` reachability question and pre-authorized the unit-test fallback if the Testing environment short-circuits the middleware; confirmed via `Program.cs` reading (`if (!app.Environment.IsEnvironment("Testing"))` wraps both the path-gate middleware and `UseHangfireDashboard`) that this is exactly the case, so `AdminDashboardAuthFilterTests.cs` was added in `QuestBoard.UnitTests` rather than attempting an HTTP-level test that could never reach the route.
- `AspNetCoreDashboardContext` requires a non-null `JobStorage` and a `RequestServices`-populated `HttpContext` to construct (confirmed via a throwaway reflection probe against Hangfire.Core 1.8.23 before writing the test) — used `NSubstitute.Substitute.For<JobStorage>()` and an empty `ServiceCollection().BuildServiceProvider()` since `Authorize()` never touches storage or DI, only `GetHttpContext()`.
- `EditProfileViewModel.IsDungeonMaster` (AccountController.Edit) has no corresponding conditional markup in `Edit.cshtml` — confirmed by reading the view before writing assertions. Rather than assert on a body marker that doesn't exist (which would be a false requirement invented by the test), the Edit tests assert 200 OK per role to prove `GetEffectiveGroupRoleAsync` resolves without throwing for DM/Admin/Player.
- Discovered and fixed a live regression during the full-suite gate run (Task 2): `QuestController.Index` (migrated in Plan 02) called `activeGroupContext.RequireActiveGroupId()` unconditionally inside the `userEntity != null` branch, which throws for SuperAdmin since `ActiveGroupContextService` intentionally returns `null` for that role. This reintroduced the exact failure mode `GroupSessionMiddlewareIntegrationTests.SuperAdmin_NoActiveGroup_NotRedirectedByMiddleware` (a pre-existing Phase 31 test) was written to catch. Verified in isolation (`dotnet test QuestBoard.IntegrationTests --filter FullyQualifiedName~GroupSessionMiddlewareIntegrationTests` failed before the fix, independent of any new test added by this plan) before fixing — this is a Rule 1 auto-fix (broken behavior caused by this phase's own prior plan, blocking the full-suite-green gate this task's `<verify>` requires).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed QuestController.Index throwing for SuperAdmin with no ActiveGroupId**
- **Found during:** Task 2 (full-suite gate run)
- **Issue:** `RequireActiveGroupId()` was called unconditionally for any authenticated user in `Index`, but `ActiveGroupContextService` intentionally returns `null` `ActiveGroupId` for SuperAdmin (system-wide access, no group scope). This threw `InvalidOperationException`, surfaced as a 500 (translated to 404 by the exception handler in tests), and broke the pre-existing `GroupSessionMiddlewareIntegrationTests.SuperAdmin_NoActiveGroup_NotRedirectedByMiddleware` test from Phase 31.
- **Fix:** `Index` now checks `User.IsInRole("SuperAdmin")` first and short-circuits to `GroupRole.Admin` without calling `RequireActiveGroupId()`, mirroring `GetEffectiveGroupRoleAsync`'s own SuperAdmin bypass (which never uses the `groupId` argument for SuperAdmin).
- **Files modified:** `QuestBoard.Service/Controllers/QuestBoard/QuestController.cs`
- **Verification:** `dotnet test QuestBoard.IntegrationTests --filter "FullyQualifiedName~GroupSessionMiddlewareIntegrationTests"` passes (9/9); full suite passes (349/349).
- **Committed in:** `acd870d` (Task 2 commit)
