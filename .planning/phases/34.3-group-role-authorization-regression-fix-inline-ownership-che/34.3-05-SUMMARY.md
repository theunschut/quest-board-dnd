# Phase 34.3 Plan 05: Dead Admin-Role Clause Deletion Summary

**Deleted three permanently-false `IsInRole("Admin")`/`IsInRoleAsync(..., "Admin")` checks — two Hangfire gates simplified to SuperAdmin-only, one redundant inner check removed from AdminResetPasswordAsync so real Admins can reset passwords again**

## Accomplishments
- Hangfire path-gate in `Program.cs` now checks `SuperAdmin` only — the dead `Admin` clause (permanently false since Phase 27 moved per-group roles off `AspNetUserRoles`) is gone
- `AdminDashboardAuthFilter` (twin Hangfire dashboard gate) simplified identically
- `IdentityService.AdminResetPasswordAsync` no longer performs a redundant `IsInRoleAsync(adminEntity, "Admin")` check that always failed post-Phase-27 — real Admins reaching the action via the `AdminOnly` policy are no longer blocked

## Task Commits

Each task was committed atomically:

1. **Task 1: Simplify both Hangfire gates to SuperAdmin-only (D-01)** - `e93cf8e` (fix)
2. **Task 2: Remove the redundant Admin check from IdentityService.AdminResetPasswordAsync (24th site)** - `a0ca005` (fix)

_No TDD tasks in this plan; both tasks were single-commit clause deletions._

## Files Created/Modified
- `QuestBoard.Service/Program.cs` - Hangfire inline middleware role gate: `if (!context.User.IsInRole("Admin") && !context.User.IsInRole("SuperAdmin"))` → `if (!context.User.IsInRole("SuperAdmin"))`
- `QuestBoard.Service/Authorization/AdminDashboardAuthFilter.cs` - Twin Hangfire dashboard gate: same clause deletion
- `QuestBoard.Repository/IdentityService.cs` - `AdminResetPasswordAsync` guard: `if (adminEntity == null || !await userManager.IsInRoleAsync(adminEntity, "Admin"))` → `if (adminEntity == null)`

## Decisions Made
- Hangfire's two gate sites were simplified to SuperAdmin-only rather than migrated to the D-02 shared `GetEffectiveGroupRoleAsync` helper — migrating would have granted group Admins new access to background-job infrastructure, reversing Phase 29's explicit SuperAdmin-only decision. Both files were verified post-edit to contain no `IActiveGroupContext` or `GetEffectiveGroupRoleAsync` references.
- `IdentityService.AdminResetPasswordAsync`'s inner Admin check was deleted outright rather than migrated to GroupRole logic — `IdentityService` lives in the Repository layer, which cannot reach the Service-layer `IActiveGroupContext`. The outer `[Authorize(Policy = "AdminOnly")]` on `AdminController` (confirmed at line 20, using the already-fixed `AdminHandler`) is the real, authoritative gate; the inner check was provably redundant dead code.
