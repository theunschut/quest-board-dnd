### Shared helper (D-02) — `QuestBoard.Domain/Interfaces/IUserService.cs` + `UserService.cs`

**Analog:** `QuestBoard.Service/Authorization/AdminHandler.cs` (SuperAdmin-bypass + role-lookup logic to port), combined with existing `IUserService.GetGroupRoleAsync` signature convention.


**Existing implementation to mirror** (`QuestBoard.Domain/Services/UserService.cs` lines 54-60):
```csharp

{
    var userId = await identityService.GetUserIdAsync(user);
    if (userId == null) return null;
    return await repository.GetGroupRoleAsync(userId.Value, groupId);
}
```

**Logic to port from** (`QuestBoard.Service/Authorization/AdminHandler.cs` lines 12-36 — the exact SuperAdmin-bypass + role-check shape the new helper must replicate verbatim):
```csharp
protected override async Task HandleRequirementAsync(
    AuthorizationHandlerContext context,
    AdminRequirement requirement)
{
    // Step 1: SuperAdmin bypass — reads claims directly, no DB call
    if (context.User.IsInRole("SuperAdmin"))
    {
        context.Succeed(requirement);
        return;
    }

    // Step 2: Null group guard
    if (activeGroupContext.ActiveGroupId is not { } groupId)
    {
        context.Fail();
        return;
    }

    // Step 3: Group role check
    var role = await userService.GetGroupRoleAsync(context.User, groupId);
    if (role == GroupRole.Admin)
        context.Succeed(requirement);
    else
        context.Fail();
}
```

**Recommended new method shape** (per RESEARCH.md's synthesis — final naming left to planner, but the return type/parameter shape below is the strongly recommended one, matching the `Get...Async` naming convention already used twice in this file):
```csharp
// IUserService.cs — new method
/// <summary>
/// Returns the effective GroupRole for the given ClaimsPrincipal in the specified group,
/// treating SuperAdmin as an automatic Admin-equivalent bypass (no group membership required).
/// </summary>
Task<GroupRole?> GetEffectiveGroupRoleAsync(ClaimsPrincipal user, int groupId);

// UserService.cs — new implementation
/// <inheritdoc/>
public async Task<GroupRole?> GetEffectiveGroupRoleAsync(ClaimsPrincipal user, int groupId)
{
    if (user.IsInRole("SuperAdmin"))
        return GroupRole.Admin;

    return await GetGroupRoleAsync(user, groupId);
}
```

Callers then do (D-03 fail-hard sites): `var role = await userService.GetEffectiveGroupRoleAsync(User, activeGroupContext.RequireActiveGroupId());` and compare `role == GroupRole.Admin` / `role == GroupRole.Admin || role == GroupRole.DungeonMaster`. This keeps the null-group decision (fail-hard vs fail-soft) visible at the call site per D-03/D-04, matching Constraint 2 in RESEARCH.md.

**`GroupRole` enum** (`QuestBoard.Domain/Enums/GroupRole.cs`):
```csharp

{
    Player = 0,
    DungeonMaster = 1,
    Admin = 2
}
```
