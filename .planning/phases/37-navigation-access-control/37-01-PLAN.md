<objective>
Extend the group-context seam so any server-rendered view can ask for the active group's `BoardType`, and stand up the failing nav-visibility test class that Plan 02 will turn green. This is the interface-first foundation for the nav-gating work: it defines the contract (`IActiveGroupContext.GetBoardTypeAsync`) and the test-infrastructure hook (`MutableGroupContext.BoardType`) that Plan 02's layout edits and their tests both depend on.

Purpose: `_Layout.cshtml`/`_Layout.Mobile.cshtml` render on every request regardless of controller, so Phase 36's per-action `ViewBag.BoardType` threading does not reach them. Extending `IActiveGroupContext` is the chosen mechanism (over Session-mirroring) specifically because the integration-test suite replaces `IActiveGroupContext` wholesale with `MutableGroupContext` and never touches `HttpContext.Session` — a Session mirror would be invisible to every existing test helper.
Output: An async `GetBoardTypeAsync` member on the context interface + production service, a settable `BoardType` on the test double, and a red `LayoutNavigationTests` class.
</objective>
