<objective>
Add a third `Dead` value to the `CharacterStatus` enum and wire it through every surface that already displays or edits character status: the Create/Edit dropdowns (auto-populated), the Details/Index status badges (desktop + mobile), the Retire/Reactivate toggle button (hidden when Dead), and the card/row dimming treatment. This extends an existing status field — it adds no new controller actions, pages, ViewModels, or EF Core migration.

Purpose: Let DMs and players mark a character as permanently deceased, visually distinct from Retired, so a dead character is removed from quest signup (already handled by the existing `Status == Active` eligibility filter — zero code change) and clearly flagged in the guild views.

Output: 1 enum member added, 2 CSS files extended with Dead styling, 4 Razor views extended with a Dead badge branch (and Details views additionally hide the toggle button when Dead).
</objective>
