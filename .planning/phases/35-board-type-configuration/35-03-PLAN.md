<objective>
Add the Board Type UI to the three Platform Group view families (desktop + `.Mobile` for each): a required dropdown with a blank placeholder and permanence warning on Create, a disabled read-only dropdown on Edit, and a badge column/line on Index.

After this plan, a SuperAdmin can pick a board type on the Create form (blank-first, must choose), see it locked on Edit, and see a per-group badge on the list — desktop and mobile. This completes the phase's human-facing behavior; the server-side contract and its automated tests were already made green in Plan 02.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SuperAdmin browser -> Platform Group forms | The disabled Edit dropdown and the Create dropdown render inside authenticated SuperAdmin-only views. Client-side `disabled` is a UX affordance, not the security boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-01 | Tampering | Edit view `<select disabled>` for BoardType (mass-assignment / over-posting, D-06) | mitigate | The disabled select does not postback its value, and NO hidden `<input asp-for="BoardType">` is added, so a normal form submit carries no BoardType. The real boundary is server-side (Plan 02: Edit POST never reads `model.BoardType`), already proven green by `EditGroup_PostingChangedBoardType_ShouldBeSilentlyIgnored`. This plan's obligation is to NOT reintroduce the tamper surface via a hidden field. (ASVS V5) |
| T-35-06 | Tampering | Create view Board Type dropdown — client removes the blank placeholder or submits an out-of-range value | accept | Server-side `[Required]` on the nullable `BoardType?` (Plan 02) rejects empty submissions; ASP.NET Core enum model binding rejects non-numeric/out-of-range values into a nullable enum as a validation failure. The view's blank-first option is a UX aid, not the validation boundary. No new server code needed here. |
| T-35-07 | Information Disclosure | Board Type badge on Index | accept | The badge only reveals a group's board type to a SuperAdmin who already administers all groups (`[Authorize(Policy = "SuperAdminOnly")]`). No sensitive data exposed. |
| T-35-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs in this phase. Nothing to verify. |
</threat_model>
