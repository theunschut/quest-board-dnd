<objective>
Add the missing object-level authorization check to `GroupPickerController.SelectGroup` (D-04): it currently validates only that the posted `groupId` exists, never that the authenticated caller is a member of it — an IDOR gap. Verify membership via the existing `IUserService.GetGroupRoleByIdAsync(userId, groupId)` primitive (null = not a member) and return 404 Not Found on non-membership (D-05, this codebase's established "hide existence, never 403" convention). SuperAdmin keeps its any-group bypass.

Purpose: A user could otherwise set any existing group as their active group by POSTing its id directly, bypassing the picker's own membership-scoped listing. This closes the same root-cause pattern Phase 49 fixed twice elsewhere.
Output: `SelectGroup` with a membership check (mirroring `Index`'s own `isSuperAdmin`/`userId` resolution style), `IUserService` injected into the controller, and a new integration test asserting 404 for a non-member.
</objective>
