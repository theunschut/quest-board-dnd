<objective>
Wire the shared Markdown editor (`_MarkdownEditor.cshtml`) into the Character **Description** and **Backstory** fields on all four Character write forms (`Create.cshtml`, `Create.Mobile.cshtml`, `Edit.cshtml`, `Edit.Mobile.cshtml`), mechanically repeating what Phase 66/67 did for Quest fields. Also strengthen the global editor text-color override in `markdown-editor.css` so it survives Character's mobile-form CSS, which uses a `:not(.badge)` catch-all Quest never had.

Purpose: Satisfies the write side of CHARMD-01 and CHARMD-02 — users get the toolbar, Preview toggle, and paragraph-break hint on Description and Backstory exactly as on Quest Description, with no new editor infrastructure.
Output: Four updated write-form views + one hardened global CSS rule; Description/Backstory editing reaches parity with the shipped Quest editor across desktop and mobile.

⚠ CRITICAL CORRECTION TO 68-PATTERNS.md — the editor scripts are NOT global. `68-PATTERNS.md` states "No Scripts-section change needed … the EasyMDE CDN `<script>`/`<link>` tags and the `markdown-editor.js` init call live in `_Layout.cshtml`/`_Layout.Mobile.cshtml` globally." This is WRONG and was confirmed wrong by direct inspection. Only `markdown-editor.css` is loaded in the layouts. The EasyMDE CDN assets, the `window.markdownAntiforgeryToken` value that `markdown-editor.js` needs to call `POST /markdown/preview`, and `markdown-editor.js` itself all live in `~/Views/Quest/_QuestFormScripts.cshtml`, which every Quest write form pulls in via its `@section Scripts`. Character forms currently pull in NONE of it, so dropping `_MarkdownEditor` in alone would render a dead plain `<textarea>` with no toolbar and no Preview. The shipped precedent for a non-Quest-controller form needing the editor is Phase 67's QuestLog `EditRecap.cshtml`/`EditRecap.Mobile.cshtml`, which include `~/Views/Quest/_QuestFormScripts.cshtml` by full path. Each Character write form must do the same — see Task 1/Task 2.
</objective>
