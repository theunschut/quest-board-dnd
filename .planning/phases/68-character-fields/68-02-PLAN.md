<objective>
Render Character **Description** and **Backstory** as formatted, sanitized HTML on the two Character Details read views (`Details.cshtml` desktop, `Details.Mobile.cshtml` mobile) by routing both fields through the existing `Html.Markdown()` helper, replacing today's raw `@Model.X` inside a line-break-preserving `<p>`. Add one companion CSS edit so Markdown list items render at full opacity on the mobile card.

Purpose: Satisfies the read side of CHARMD-01 and CHARMD-02 — Description and Backstory display as formatted HTML wherever they appear, with no doubled spacing (the old plain-text `white-space: pre-wrap` is dropped; `.markdown-content { white-space: normal }` from Phase 67 takes over globally).
Output: Two updated read views + one CSS catch-all extension. No email surface (Character fields never appear in an email — confirmed in CONTEXT.md), no card/list preview (the roster Index shows no Description/Backstory snippet — confirmed), and no Manage page (Characters have none).

Note: `Html.Markdown()` emits block-level HTML (`<h1-6>`, `<ul>`, `<blockquote>`) that cannot legally sit inside a `<p>`. The current markup wraps both fields in `<p class="form-control-plaintext" style="white-space: pre-wrap;">` (desktop) and `<p class="character-info-value mb-0">` (mobile). Both `<p>` wrappers are replaced by a direct `Html.Markdown()` call inside the existing non-`<p>` container, matching the block-safe `<div>` precedent the Quest Details views and email templates already use.

Import note: `Html.Markdown` is an `internal static` **extension method** in namespace `QuestBoard.Service.Extensions`. The root `Views/_ViewImports.cshtml` does NOT import that namespace, so BOTH Character Details views MUST add `@using QuestBoard.Service.Extensions` to their per-view top import block — this is exactly what every shipped `@Html.Markdown` call site already does (`Views/Quest/Details.cshtml:5`, plus Quest `Details.Mobile`, `Manage`, `Manage.Mobile`, and both `QuestLog/Details` views; documented in 66-06-SUMMARY.md). Without the import, `Markdown` will not resolve on `IHtmlHelper<CharacterViewModel>` and `dotnet build` fails.

Heading-color is already correct on both platforms and needs NO new CSS — verified in 68-UI-SPEC.md (Typography → "heading color … resolved for both platforms"): `site.css`'s `.modern-card h1-h6/p/li/span/small { color:#F4E4BC !important }` rules win by `!important` on desktop regardless of wrapper depth, and the mobile catch-all already colors `<p>`/`<a>` — the ONLY mobile gap is `<li>`, which Task 2 closes. Do NOT port Quest/Manage's `.modern-card-body > .markdown-content h1..h6 { color:#1a1a1a }` override into Character Details — it requires a direct-child relationship that does not hold here (Character Details nests `.markdown-content` one level deeper inside a `<div class="mb-3">` label wrapper) and would be dead CSS.
</objective>

From QuestBoard.Service/Extensions/HtmlHelperExtensions.cs:
- `internal static IHtmlContent Markdown(this IHtmlHelper html, string? markdown)` in `namespace QuestBoard.Service.Extensions` — resolves `IMarkdownService`, calls `RenderToHtml(markdown, MarkdownRenderTarget.Web)` (the Web sanitizer profile — correct here; NOT `.Email`), and wraps the result in `<div class="markdown-content">…</div>`. That `.markdown-content` wrapper carries its own typography (heading scale, blockquote, `white-space: normal`), so the new call site needs no extra class of its own.

Call as `@Html.Markdown(Model.Description)` / `@Html.Markdown(Model.Backstory)`.

**Per-view `@using` is REQUIRED (do not skip):** `Markdown` is an extension method, so C# resolves it only when its containing namespace `QuestBoard.Service.Extensions` is in `@using` scope. The root `Views/_ViewImports.cshtml` imports `QuestBoard.Domain.*`, `QuestBoard.Service`, and the `QuestBoard.Service.ViewModels.*` namespaces — but NOT `QuestBoard.Service.Extensions`. Consequently every shipped `@Html.Markdown` call site adds `@using QuestBoard.Service.Extensions` at the top of its own view (`Views/Quest/Details.cshtml:5`, `Views/Quest/Details.Mobile.cshtml`, `Manage.cshtml`, `Manage.Mobile.cshtml`, `Views/QuestLog/Details.cshtml`, `Views/QuestLog/Details.Mobile.cshtml`; see 66-06-SUMMARY.md). Both Character Details views currently import ONLY `@using QuestBoard.Domain.Enums` and `@using QuestBoard.Service.ViewModels.CharacterViewModels`, so each task below must add the `@using QuestBoard.Service.Extensions` line. Omitting it makes `dotnet build` fail with `IHtmlHelper<CharacterViewModel>` does not contain a definition for `Markdown`.
