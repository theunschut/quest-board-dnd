Why the integration-test gate matters here specifically: Character write forms now include a partial that lives in a DIFFERENT controller folder (`~/Views/Quest/_QuestFormScripts.cshtml`). Razor partial resolution is a runtime check, not a compile-time one — a wrong path builds clean but throws `InvalidOperationException: The partial view … was not found` the moment the view renders. `CharactersControllerIntegrationTests` GETs/renders Create, Edit, and Details, so it will surface any such regression before the operator ever loads a page.
