# Phase 40 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — server-rendered ASP.NET Core Razor MVC + Bootstrap 5, no npm/React/Vite frontend toolchain. shadcn is not applicable to this stack. |
| Preset | not applicable |
| Component library | Bootstrap 5 (bundled via `_Layout.cshtml`/`_Layout.Platform.cshtml`), no component framework beyond it |
| Icon library | Font Awesome (`fas fa-*`, always paired with `me-2`/`me-1` spacing per CLAUDE.md convention) |
| Font | Body/UI text: Bootstrap default system font stack (no override). Display/heading/brand text: `'Cinzel', serif` (Google Fonts, loaded in `_Layout.Platform.cshtml`), applied via `.modern-card h1–h6`, `.navbar-brand`. |

This phase is a **pure redesign within an existing, mature design system** — no new visual language is introduced. Every token below documents what already exists in `site.css`/`platform-group.mobile.css` and is verified directly from the codebase, not proposed. All values are prescriptive defaults for this phase; there were no open design questions requiring user input because CONTEXT.md's D-01 through D-09 already lock every layout/interaction decision, and this phase only needs to match established visual conventions.

## Spacing Scale

Declared values (must be multiples of 4, mapped to this codebase's Bootstrap utility classes):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | `me-1`/`gap-1` — icon-to-label spacing inside badges/buttons |
| sm | 8px | `mb-2`/`gap-2` — per-row inline control spacing (role select + Add button) |
| md | 16px | `mb-3`/`p-3` — default form-group and card spacing (existing `.mb-3` convention) |
| lg | 24px | `py-4`/`gap-4` between the two Members-page columns on desktop |
| xl | 32px | `mb-4`/`py-5` — section breaks, empty-state vertical padding (matches existing `.py-5` empty state) |
| 2xl | 48px | not used in this phase |
| 3xl | 64px | not used in this phase |

Exceptions:
- Per-row inline form controls (role `<select>` + Add button) use `gap-2` (8px) between them, matching Pattern 2's `d-inline-flex align-items-center gap-2` — an established codebase pattern, not a new exception.
- Column gutter between the left (Members) and right (Available Users) columns on desktop: use Bootstrap grid `g-4` (24px) on the wrapping `.row`, consistent with `lg` token above.

## Typography

This phase adds no new type scale — it reuses the two type roles already established by `.modern-card`/Bootstrap defaults. No new sizes or weights are introduced.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (Bootstrap `1rem` default) | 400 (500 inside `.modern-card p/li/span/small` per existing override) | 1.5 |
| Label | 16px (`.form-label`, Bootstrap default) | 600 (`.modern-card .form-label` existing override) | 1.5 |
| Heading (h5 — column titles, modal title) | 20px (Bootstrap `h5` ≈ `1.25rem`) | 600 (`.modern-card h1–h6` existing override) | 1.2 |
| Display (h2 — page title "Members — {Group.Name}") | 28px (Bootstrap `h2` ≈ `1.75rem`) | 600, font-family `'Cinzel', serif` per `.modern-card-header h2` | 1.2 |

No 3rd/4th size is needed beyond these four — the existing view already uses exactly this scale (`h2` page title, `h5` section subheadings, body/label text, no other sizes).

## Color

This phase inherits the existing "Fantasy Quest Board" palette in full — no new colors are introduced. The whole app uses a full-bleed background image with translucent glass "modern-card" panels (backdrop-filter blur), so the 60/30/10 split below maps to this visual language rather than flat fills.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Full-page background image (`/images/Notice Board-blank2.jpg`, fixed/cover) showing through translucent glass panels | Page backdrop behind both columns |
| Secondary (30%) | `rgba(255, 255, 255, 0.15)` glass card fill + `rgba(255, 255, 255, 0.05)` header fill (`.modern-card`, `.modern-card-header`) with `backdrop-filter: blur(15px)` and `1px solid rgba(255, 255, 255, 0.3)` border | Both column containers (Members card, Available Users card), and the Create New User modal content (`.modal-content`) |
| Accent (10%) | `#ffc107` (Bootstrap warning/gold) for primary interactive emphasis; `#FFD700` reserved for hover-state brand/heading glow only | Reserved for: the "Create New User" trigger button (`btn-warning`, matching `CreateUser.cshtml`'s existing `btn-warning` "Create User" submit button), the search "Search" submit button border/focus states, and focus rings on `.form-control:focus`/`.form-select:focus` (`border-color: #ffc107`). Do not apply gold to more than these interaction points — body text, badges, and secondary buttons must stay off-accent. |
| Destructive | `#dc3545` (Bootstrap `btn-danger`/`alert-danger`) | Reserved exclusively for the existing "Remove Member" button — unchanged by this phase. The new per-row "Add" action is NOT destructive and must use `btn-success` (`#27ae60`/`#219a52` gradient per `.modern-card .btn-success`), matching today's "Add to Group" button color. |

Parchment text color `#F4E4BC` (with dark text-shadow) is the established body/heading text color inside `.modern-card` — applies automatically via existing CSS, no new class needed.

Additional semantic colors already established and reused as-is (no new colors):
- Role badges: Admin = `bg-danger` (`fa-shield-alt`), DungeonMaster = `bg-warning text-dark` (`fa-crown`), Player = `bg-primary` (`fa-dice-d20`) — unchanged from today's Members table, and the same badge set should visually anchor the inline role `<select>` on each Available Users row (the select's options are plain text; no badge styling needed on the select itself).
- Alert colors: `alert-success` (green), `alert-warning` (amber), `alert-danger` (red) — reused verbatim for the three-outcome flash messages from `CreateOrAddToGroupAsync` (D-07).

## Layout Notes (supplementing the contract above)

Captured here because this phase's core requirement (MEMBERS-01/02/03) is structural, not just visual — the checker and executor need this to validate against the locked CONTEXT.md decisions:

- **Desktop (`Members.cshtml`):** Two-column Bootstrap grid (`row g-4`) inside the existing single `.modern-card`, OR two side-by-side `.modern-card` panels — either is acceptable as long as both columns are visually distinct sections with their own `modern-card-header`-style subheading (`h5` with icon, matching `<h5><i class="fas fa-user-plus text-danger me-2"></i>Add Member</h5>` precedent, retitled appropriately e.g. "Current Members" / "Available Users"). Left column = current members table (unchanged markup). Right column = search form (D-01) + available-users table with per-row inline Add controls (D-03) + "Create New User" button opening the modal (D-05).
- **Mobile (`Members.Mobile.cshtml`):** Single vertical stack inside the existing `.platform-group-card-mobile` — Members cards first (unchanged), then search input + Available Users `.group-card-mobile` cards each with an inline role select + Add button, then the Create New User button/modal last (D-08). The Create New User modal markup is reused identically on mobile, unchanged (D-09/Pattern 4) — Bootstrap modals are already responsive elsewhere (`ShopManagement.Mobile.cshtml`).
- **Create New User modal:** Structure mirrors `ShopManagement/Index.cshtml`'s `#denyModal` exactly — `modal fade` → `modal-dialog` → `modal-content bg-dark text-light` → `<form method="post">` with `@Html.AntiForgeryToken()`, `modal-header border-secondary` / `modal-body` / `modal-footer border-secondary`. Do NOT use the `.modern-card` glass style inside the modal — the codebase's only modal precedent (`ShopManagement`) uses solid `bg-dark text-light`, not the glass/parchment card style; matching that precedent takes priority over matching the page's outer glass panels.
- **Per-row Add form:** `d-inline-flex align-items-center gap-2` layout per Pattern 2 — hidden `UserId` input, `Role` `<select class="form-select form-select-sm">` defaulting to `Player`, `btn-sm btn-success` submit button. Antiforgery token per row via `asp-antiforgery="true"`.
- **Search form:** `<form method="get">` GET round-trip (D-01), `<input type="text" name="search" class="form-control" value="@Model.SearchQuery">` + `<button type="submit" class="btn btn-warning">` (gold accent, since Search is this column's primary action) with `fas fa-search me-1`.
