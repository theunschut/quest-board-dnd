# Phase 66 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope note:** This is a proof-of-concept phase wiring a third-party Markdown editor (EasyMDE) into an existing Bootstrap 5 MVC app with an established "fantasy/parchment" visual theme. Almost every visual decision was already locked by `66-CONTEXT.md` (D-01 through D-09) and `66-RESEARCH.md`. This contract's job is mostly to **pre-populate those locked decisions in one place** and to fill the few genuinely open gaps (rendered-Markdown typography, toolbar fit-at-320px, link/blockquote treatment) that neither upstream artifact addressed. No new color palette, no new component library, no shadcn — this app is not a React/Next.js/Vite project, so the shadcn gate does not apply.

## Design System

| Property | Value |
|----------|-------|
| Tool | none — Bootstrap 5 MVC app (no `components.json`, not a React/Next.js/Vite project; shadcn gate not applicable) |
| Preset | not applicable |
| Component library | Bootstrap 5 (bundled, pre-existing) + EasyMDE 2.21.0 (new this phase, CDN + SRI, zero build step) |
| Icon library | Font Awesome 6.4.0 (`all.min.css`, cdnjs, pre-existing) + Font Awesome v4-shims 6.4.0 (new this phase — required so EasyMDE's default `fa fa-bold`-style toolbar icons render; see RESEARCH.md Pitfall 3) |
| Font | `'Cinzel'` serif (decorative headings/branding, pre-existing, Google Fonts) + system default sans-serif stack (body text, Bootstrap default, no override found in `site.css`) |

**Visual anchor:** The rendered Markdown content (`.markdown-content`) remains the primary visual anchor on Details, Manage, and the board card — it's what a player is there to read. The EasyMDE toolbar and editor chrome on the Create/Edit/Follow-Up forms are secondary, deliberately unstyled per D-03, and must not compete visually with the description text itself.

## Spacing Scale

Declared values (must be multiples of 4) — this app's existing Bootstrap-derived spacing already follows this scale; unchanged by this phase:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`me-1`), inline padding |
| sm | 8px | Compact element spacing (`me-2`) |
| md | 16px | Default element spacing (`mb-3`) |
| lg | 24px | Section padding (`mb-4`, `.modern-card-body` padding) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Exceptions (new this phase):**

- **44px min-width / min-height on `.editor-toolbar button`** — EDITOR-06's locked touch-target floor. Required as an explicit CSS override because the app's existing global 44px rule (`mobile.css`) only targets `.btn`, `input`, `select`, `textarea`, `.form-control`, `.form-select` — EasyMDE's toolbar `<button>` elements carry none of those classes, so the existing rule does not apply automatically (confirmed in RESEARCH.md Pitfall 4).
- **0px gap between toolbar buttons** — grid-compliant (0 is a multiple of 4) and the only value that satisfies EDITOR-06's locked "one row, no overflow/scroll mechanism" requirement at this button count/size. Math: 7 buttons (Bold/Italic/Heading/List/Link/Quote/Preview) × 44px = 308px, leaving 12px of slack inside the 320px viewport budget (iPhone SE 1st-gen class, narrowest realistic mobile CSS width) at 0px gap. A non-zero gap was considered and rejected: even the smallest standard spacing token (4px × 6 gaps = 24px) pushes the total to 332px, overflowing 320px. Override EasyMDE's default toolbar button margin to `0` to enforce this.
- **0 extra horizontal padding on the `.editor-toolbar` wrapper itself** — override EasyMDE's own default toolbar padding to reclaim every pixel toward the 320px budget above.
- **Verification still required, not optional:** build the toolbar, then explicitly test at a real 320px-wide viewport (Chrome DevTools narrowest preset or an actual small device) before treating EDITOR-06 as satisfied. Even though the 0px-gap arithmetic leaves 12px of slack, browser rendering quirks (button borders, box-sizing, EasyMDE's own internal button padding) can consume that margin — if it still overflows, **do not** silently shrink buttons below 44px or drop a button to make it fit — both would violate a locked requirement. Escalate as a blocking finding instead (this mirrors RESEARCH.md's own Open Question #1, carried forward here rather than papered over).
- 20px (1.25rem) box padding on dark-overlay content boxes (`.quest-description-box` and the new Manage-page collapsible section body) — pre-existing convention (`quests.css:818`), unchanged by this phase.

## Typography

New typography surface introduced by this phase: rendered Markdown output (`.markdown-content` — headings, paragraphs, lists, blockquotes, links, bold/italic) appearing for the first time on Details, Manage, and the board card. Nothing in the existing codebase renders arbitrary user-authored heading levels today, so this table locks a scoped, restrained scale rather than letting the browser/site defaults apply unchecked.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (1rem) | 400 regular | 1.6 |
| Label | 16px (1rem) | 700 bold | 1.2 |
| Heading | 20px (1.25rem) | 700 bold | 1.2 |
| Display | not used | — | — |

**Notes (defaults set by this research, not pre-answered upstream):**

- **Markdown H1–H6 collapse to a single visual tier (the "Heading" row above), scoped to `.markdown-content h1, .markdown-content h2, ... h6`.** Quest Description is short-form card/detail content, not a document — an unbounded H1 rendering at the browser default (~32px) inside a quest card would visually overpower the surrounding UI. This explicitly overrides the app's global `h1, h2, h3, h4, h5, h6 { color: #1a0f08; ... }` rule (`site.css:586`), which was designed for page titles on the light parchment background, not nested dark-overlay boxes — using it unscoped would render markdown headings in near-invisible dark brown against `.quest-description-box`'s black overlay (`rgba(0,0,0,0.4)`). Set `.markdown-content h1`–`h6` to `color: #F4E4BC` (parchment gold, matching `.quest-description-box`'s existing text color) with the same text-shadow already used by that box.
- **Strong/bold emphasis** inside rendered Markdown uses the same 700 weight as Heading (reuses the two-weight budget: 400 body / 700 emphasis+heading+label).
- **Blockquote:** `border-left: 3px solid rgba(255, 193, 7, 0.5)` (accent gold at reduced opacity, echoing `.quest-description-box`'s existing `rgba(255, 193, 7, 0.3)` border convention), `padding-left: 12px`, `font-style: italic`, body color/size otherwise unchanged.
- **Links** inside rendered Markdown use Bootstrap's default, unmodified link styling (`var(--bs-link-color)`, underlined) — no override. Accent gold is intentionally *not* reused for links (see Color section — accent is reserved for specific chrome elements, not general interactive/body-content elements), and no existing `a`/`a:hover` override exists in `site.css` to conflict with.
- Editor-chrome text (toolbar button `title` tooltips, the Bootstrap hint tooltip) is EasyMDE/Bootstrap default sizing — not restyled, per D-03.

## Color

No new palette — this phase reuses the app's existing fantasy/parchment theme exactly as established. Values below are the existing tokens, documented here so the checker/executor don't need to re-derive them from CSS.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Notice Board background image + `rgba(0, 0, 0, 0.4)` dark overlay panels | Page background, `.quest-description-box`, `.recap-display-box` — pre-existing, unchanged |
| Secondary (30%) | `rgba(255, 255, 255, 0.15)` glass overlay | `.modern-card`, `.modern-card-header` — pre-existing, unchanged |
| Accent (10%) | `#FFC107` (Bootstrap warning / `text-warning`) | See "Accent reserved for" below |
| Destructive | not applicable this phase | No delete/remove/destructive actions introduced by Phase 66. Existing app convention (`#dc3545` / `btn-danger`) is unused here. |

**Accent reserved for (this phase's new UI only):**
- The Manage-page collapsible Description section's toggle button + chevron icon (`fas fa-chevron-down`), matching `_QuestSection.cshtml`'s existing `btn-link text-decoration-none text-warning fw-bold` pattern exactly.
- The Manage-page collapsible Description section's header icon (`fas fa-scroll text-warning`).
- The paragraph-break hint icon (`fas fa-info-circle text-warning`, EDITOR-05).
- The blockquote left-border accent inside rendered Markdown content (`rgba(255, 193, 7, 0.5)`, see Typography).
- **NOT** used to restyle EasyMDE's own toolbar buttons or icons — D-03 locks the default out-of-the-box toolbar appearance; accent gold does not bleed into the third-party widget's chrome.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | No new CTA this phase. Existing form submit buttons unchanged: "Create Quest" (`btn-warning`, `fa-plus`, Create), "Update Quest" (`btn-warning`, `fa-save`, Edit), "Save Follow-Up Quest" (`btn-success`, `fa-scroll`, CreateFollowUp). The Preview toggle uses EasyMDE's own default eye-icon button with its native title tooltip ("Toggle Preview") — not customized, per D-03. |
| Empty state heading | Not applicable — Description is a required field (red-asterisk validation, `asp-validation-for` on all 3 write forms). No quest ever has an empty Description to render. |
| Empty state body | Not applicable (see above). The Manage-page collapsible Description section is collapsed by default regardless of content length (D-02) — that's a visibility default, not an empty-state condition. |
| Error state | Preview round-trip failure (network error or non-2xx from `POST /markdown/preview`): render `<p class="text-danger">Preview failed to load.</p>` inside the preview pane (locked verbatim in `66-RESEARCH.md` Pattern 1). Rest of the editor stays usable — user can retry Preview or continue editing/saving normally. |
| Destructive confirmation | None — Phase 66 introduces no delete/remove/destructive actions. |
| Paragraph-break hint (tooltip, EDITOR-05) | "Supports Markdown formatting. Leave a blank line between paragraphs." — exact wording locked by D-09. Bootstrap tooltip, triggered on hover (desktop) / tap (mobile). |
| Manage-page Description section header (D-02, new surface) | Title: "Description". Icon: `fas fa-scroll` (mirrors the icon already used for quest/description context on Create/CreateFollowUp headers). Collapsed by default on both desktop and mobile. Toggle chevron: `fas fa-chevron-down`, following `_QuestSection.cshtml`'s exact structural pattern (card-header + `btn-link text-warning fw-bold` + collapse target). |
