# Phase 34.2 Plan 05: ActiveGroupId Null-Guard + Verification-Gap Tests Summary

**Adds a fail-fast `RequireActiveGroupId()` Domain guard (ASVS V4 Access Control) and closes the four remaining CONCERNS.md verification gaps — AutoMapper enum-cast round-trip, Group Query Filter non-existent-group enforcement, and Hangfire retry policy — without touching the existing null-is-valid query filter contract.**

## Accomplishments
- Added `ActiveGroupContextExtensions.RequireActiveGroupId()` in the Domain layer — a pure, opt-in fail-fast guard that throws `InvalidOperationException` when `ActiveGroupId` is unexpectedly null, converting the "silent null-scope" scaling risk (CONCERNS.md) into an explicit, testable error for request-scoped callers that require a concrete group. Verified the `QuestBoardContext.HasQueryFilter` lambda's null-see-all predicate is byte-for-byte unchanged.
- Proved every `EntityProfile.cs` enum cast (`SignupRole`, `VoteType`, `ItemType`, `ItemRarity`, `ItemStatus`, `TransactionType`, `CharacterStatus`, `CharacterRole`, `DndClass`, `GroupRole`) round-trips all 42 declared enum values without drift, using a cast-invariant assertion pattern that mirrors the exact `(int)`/`(T)` cast EntityProfile performs at each mapping site.
- Added `GroupFilter_NonExistentGroup_ReturnsEmpty` to the existing `TenantIsolationTests` — proves a non-existent `ActiveGroupId` (999) returns zero quests via the query filter, closing the "Group Query Filter Enforcement Not Tested" gap. The existing `GroupFilter_HidesQuestFromOtherGroup`, `GroupFilter_ShowsQuestFromSameGroup`, and `GroupFilter_NullGroupIdShowsAllGroups` tests are unchanged and still pass.
- Added `RequireActiveGroupId_NullContext_Throws` to the same class — an integration-level companion proving the new guard throws on null and returns on a set value, without disturbing the SuperAdmin/seeding null-see-all path.
- Added `HangfireRetryPolicyTests` — locks the real `Hangfire.AutomaticRetryAttribute` (not the nonexistent `UseAutoRetry`) at 5 attempts / 1,2,4,8,16s backoff, matching exactly what Plan 03 registered in `Program.cs`.

## Decisions Made
- Confirmed AutoMapper 16.1.1's `LicenseKey` requirement (`config.LicenseKey = builder.Configuration["AutoMapper:LicenseKey"]` in `Program.cs`, empty default in `appsettings.json`) has no equivalent wiring in any existing unit test, and no test currently constructs a real `IMapper` from `EntityProfile`. Used the plan's explicitly-sanctioned fallback: assert the `(int)`/`(T)` cast invariant directly via `Enum.GetValues<T>()` + `Enum.IsDefined`, which proves the exact fragility CONCERNS.md flags (silent truncation/wrap on an undefined int) without fighting mapper construction.
- Kept `RequireActiveGroupId()` unwired from all production call sites (controllers, `QuestBoardContext`, Hangfire jobs) per the plan's explicit scope — this plan introduces the seam and proves it works; wiring it into request-scoped controllers is left for a future phase.
- Implemented the null-guard integration test as a direct assertion against `factory.TestGroupContext` (the registered `IActiveGroupContext` singleton) rather than routing it through an HTTP request, since the guard is intentionally not called from any controller action yet.
