<objective>
Deliver the write half of PROFILEMD-02: wire the shared Markdown editor into Shop Item Description on all four ShopManagement write forms (Create, Create.Mobile, Edit, Edit.Mobile). Description is `[Required]` on both `CreateShopItemViewModel` and `EditShopItemViewModel` — the first required field this milestone has migrated — so the partial is called with `Required = true` (rendering the red asterisk), and the milestone's client-side required-field handling is exercised in earnest for the first time.

Two wrinkles unique to these forms are handled here: (1) `Create.cshtml` and `Edit.cshtml` carry bespoke inline `<script>` submit validators that read the raw `<textarea>` value directly, which EasyMDE hides — those must read the live editor content instead; (2) to make that read robust regardless of event ordering, the shared `markdown-editor.js` is extended (additively) to expose each built EasyMDE instance on its textarea element.

Purpose: Give DMs formatted-text authoring for Shop Item Description without breaking the existing 20-character client-side validation on the required field.
Output: An additive instance handle in `markdown-editor.js`; four Shop write forms rendering the EasyMDE editor for Description (Required=true) and loading `_QuestFormScripts`; and two inline validators reading the live editor value.
</objective>
