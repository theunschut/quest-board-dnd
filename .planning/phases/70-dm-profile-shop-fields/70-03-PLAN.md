Deliver the read half of PROFILEMD-02: render Shop Item Description as formatted HTML on both Shop Details read surfaces (desktop `_ShopItemDetailsContent.cshtml` — used for the full page AND the Bootstrap modal — plus mobile `Shop/Details.Mobile.cshtml`), and switch the two existing plain-text Description teasers to strip Markdown before truncating so raw syntax never leaks.

- D-02: the DM-facing ShopManagement dashboard's three identical 50-char inline substring teasers also switch to `ExtractPlainText()`-before-truncate.

Purpose: Make Description display as formatted HTML wherever it is shown in full, and keep the list/card teasers readable (plain text, no leaking `#`/`*`/`>` syntax).
Output: Two read views rendering Description via `Html.Markdown()` with the redundant inline pre-wrap removed, and two teaser surfaces deriving their preview text from `ExtractPlainText()` — the ShopManagement one via a single consolidated `DescriptionTeaser` helper.
</objective>
