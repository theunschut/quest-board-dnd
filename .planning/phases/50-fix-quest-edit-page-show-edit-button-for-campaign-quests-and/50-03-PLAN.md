Make the Edit page hide the four Campaign-irrelevant fields exactly the way the Create page already does, and supply the `ViewBag.BoardType` the view needs to branch on — including the corollary fix on the Edit POST validation-failure path that RESEARCH.md's Pitfall 3 identified as a crash risk introduced by this very change.

Output: Edited `QuestController.cs` (Edit GET + Edit POST), `Edit.cshtml`, `Edit.Mobile.cshtml`. No service/domain/data changes — the Edit POST already sanitizes the four fields server-side; that logic is untouched.
