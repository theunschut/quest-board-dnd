<objective>
Fix the iOS Safari fixed-background scroll bug (#116) by replacing `background-attachment: fixed` on `body` with a `position: fixed` `body::before` pseudo-element layer in both `mobile.css` and `site.css`.

Purpose: On iOS Safari, `background-attachment: fixed` is a known WebKit compositing limitation — the background image scrolls with page content instead of staying visually fixed. Desktop browsers and devtools "mobile" emulation do NOT reproduce it, so this can only be validated on a real iOS Safari session. This is MOBILE-01.

Output: Both global stylesheets carry a `body::before` full-viewport background layer; the broken `background-attachment: fixed` declaration is gone from both files.
</objective>
