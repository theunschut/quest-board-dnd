The test class should reset the database before each test to avoid cross-test contamination (`factory.ResetDatabase()`). After reset, seed via `factory.Database.CreateContext()` (which uses ActiveGroupId=null so it can write data to any group freely).

**Seeding approach:**
- Use `factory.Database.CreateContext()` to get a context that bypasses the filter (null = see all).
- Add a Group 2 entity (`new GroupEntity { Id = 2, Name = "OtherGroup", CreatedAt = DateTime.UtcNow }`).
- Seed a quest with `GroupId = 2` and a distinctive title like `"GroupTwoQuest"`.
- Seed a quest with `GroupId = 1` and title `"GroupOneQuest"` (use an existing or new DungeonMaster user; check how other tests create users).

**ActiveGroupId manipulation:**
- Set `factory.TestGroupContext.ActiveGroupId = 1` before requests that should see only Group 1.
- Reset to `1` (default) after each test to avoid bleeding.
- For the null test (Test 3), query via `factory.Database.CreateContext()` directly — bypasses the filter entirely.

**Test 3 implementation note:** Since the HTTP client always routes through the stub (which would be null), and null = see-all in Phase 28, an HTTP GET with null context would return both groups' quests. However, a more reliable test for null = see-all is to query via DbContext directly: `using var ctx = factory.Database.CreateContext(); var all = ctx.Quests.ToList();`. This is unambiguous and does not depend on HTTP routing. Assert `all.Count >= 2` and that both quest titles appear.

The full file structure:
```csharp
using QuestBoard.IntegrationTests.Helpers;
using QuestBoard.Repository.Entities;

namespace QuestBoard.IntegrationTests.Tests;

public class TenantIsolationTests(WebApplicationFactoryBase factory)
    : IClassFixture<WebApplicationFactoryBase>
{
    // Test 1: quest from Group 2 hidden when active group = 1
    [Fact]
    public async Task GroupFilter_HidesQuestFromOtherGroup()
    {
        factory.ResetDatabase();

        // Seed: group 2 + quest in group 2
        await using var ctx = factory.Database.CreateContext();
        ctx.Groups.Add(new GroupEntity { Id = 2, Name = "OtherGroup", CreatedAt = DateTime.UtcNow });
        // Need a DM user — add minimal user if none exists, or reuse seeded user from TestDataHelper if available
        // Seed quest with GroupId = 2
        ctx.Quests.Add(new QuestEntity
        {
            Title = "GroupTwoQuest",
            Description = "Hidden quest",
            GroupId = 2,
            DungeonMasterId = 1,  // assume seed user exists; adjust if needed based on TestDataHelper pattern
            CreatedAt = DateTime.UtcNow
        });
        await ctx.SaveChangesAsync();

        factory.TestGroupContext.ActiveGroupId = 1;
        using var client = factory.CreateClient();

        var response = await client.GetAsync("/Quest");
        var body = await response.Content.ReadAsStringAsync();

        Assert.DoesNotContain("GroupTwoQuest", body);
    }

    // Test 2: quest from Group 1 visible when active group = 1
    [Fact]
    public async Task GroupFilter_ShowsQuestFromSameGroup()
    {
        factory.ResetDatabase();

        await using var ctx = factory.Database.CreateContext();
        ctx.Quests.Add(new QuestEntity
        {
            Title = "GroupOneQuest",
            Description = "Visible quest",
            GroupId = 1,
            DungeonMasterId = 1,
            CreatedAt = DateTime.UtcNow
        });
        await ctx.SaveChangesAsync();

        factory.TestGroupContext.ActiveGroupId = 1;
        using var client = factory.CreateClient();

        var response = await client.GetAsync("/Quest");
        var body = await response.Content.ReadAsStringAsync();

        Assert.Contains("GroupOneQuest", body);
    }

    // Test 3: null = see all (raw DbContext bypasses filter)
    [Fact]
    public async Task GroupFilter_NullGroupIdShowsAllGroups()
    {
        factory.ResetDatabase();

        await using (var ctx = factory.Database.CreateContext())
        {
            ctx.Groups.Add(new GroupEntity { Id = 2, Name = "OtherGroup", CreatedAt = DateTime.UtcNow });
            ctx.Quests.Add(new QuestEntity
            {
                Title = "GroupOneVisible",
                Description = "Group 1 quest",
                GroupId = 1,
                DungeonMasterId = 1,
                CreatedAt = DateTime.UtcNow
            });
            ctx.Quests.Add(new QuestEntity
            {
                Title = "GroupTwoVisible",
                Description = "Group 2 quest",
                GroupId = 2,
                DungeonMasterId = 1,
                CreatedAt = DateTime.UtcNow
            });
            await ctx.SaveChangesAsync();
        }

        // TestDatabase.CreateContext() uses MutableGroupContext { ActiveGroupId = null } — sees all
        await using var readCtx = factory.Database.CreateContext();
        var allQuests = readCtx.Quests.ToList();

        Assert.Contains(allQuests, q => q.Title == "GroupOneVisible");
        Assert.Contains(allQuests, q => q.Title == "GroupTwoVisible");
    }
}
```
