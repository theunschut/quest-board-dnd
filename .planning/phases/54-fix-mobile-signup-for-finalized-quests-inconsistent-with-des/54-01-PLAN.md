<objective>
Change `QuestController.JoinFinalizedQuest` so a brand-new Player join on a full, finalized One-Shot quest is waitlisted (`IsSelected = false`) instead of hard-rejected, reusing the existing Phase 44 waitlist machinery unchanged (D-03). Assistant DM / Spectator joins are unaffected and stay always-seated (D-05). Because `JoinFinalizedQuest` is one shared controller action, this correction applies identically to desktop and mobile callers (D-04).

Purpose: Removes the capacity hard-wall that made "full" a dead end for new joiners, routing them into the same waitlist/auto-promotion pipeline existing signups already use. This is the backend half of the phase; the view/UI half (mobile Join card + copy) ships in plan 02 against disjoint files.
Output: An edited `JoinFinalizedQuest` action plus net-new integration coverage and an extended unit test — no new controller action, no migration, no service/repository changes.
</objective>
