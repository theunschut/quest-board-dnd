Rename the existing `ImageData` column to `OriginalImageData` on all three 1:1 image tables (`CharacterImages`, `DungeonMasterProfileImages`, `ContactImages`) and add a new nullable `CroppedImageData` column to each, via a single hand-edited EF Core migration that preserves every existing photo. This is the schema bottleneck the rest of the phase builds on.

Purpose: Establishes the dual-image storage columns without any server-side image processing, keeping every current photo intact under the semantically-correct "original" name (per user decision D-02). No view/form changes (D-04).
Output: Three renamed entities, updated AutoMapper profile, regenerated model snapshot, and one verified data-preserving migration.
</objective>
