## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Controller (multipart POST) | Untrusted CroppedPictureFile IFormFile crosses into the app; client claims it is "already cropped" |
| Controller → image read endpoint | Any authenticated user can request GetCroppedPicture / GetCroppedContactImage / GetDMProfilePicture by id |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-46-03-01 | Tampering | Client-submitted CroppedPictureFile | mitigate | Never trust the "already cropped" client claim — the cropped file is passed through the existing IImageValidationService.ValidateImagePair server-side (MIME/extension/size), identical to the original file's validation, before any bytes are persisted. Task 2 changes the call from `cropped: null` to the real cropped ImageFileInput. |
| T-46-03-02 | Information Disclosure / Elevation of Privilege | New GetCroppedContactImage endpoint | mitigate | The Contacts cropped-read action replicates GetContactImage's exact IsVisibleTo(contact, currentUser.Id, includeHidden) gate — verified by the integration test asserting NotFound for a non-visible contact. A cropped variant without this gate would leak hidden-contact images. |
| T-46-03-03 | Information Disclosure | New GetCroppedPicture (Characters) endpoint | accept | Its sibling GetProfilePicture has no visibility gate today (bare [HttpGet] under class-level [Authorize]); the cropped variant matches that exactly rather than adding an inconsistent new gate. Character images are already retrievable by any authenticated user by id under the existing design — no new exposure is introduced. |
| T-46-03-04 | Information Disclosure | DM GetDMProfilePicture repoint | mitigate | The existing IsTargetInActiveGroupAsync group-scoping gate is preserved unchanged during the repoint; only the served column changes (original → cropped-or-fallback), not the authorization boundary. |
| T-46-SC | Tampering | npm/pip/cargo installs | mitigate | No package-manager install occurs in this plan. N/A. |
</threat_model>
