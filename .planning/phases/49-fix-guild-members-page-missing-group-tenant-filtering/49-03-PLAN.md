<objective>
Harden `UserTransaction`'s currently-incidental group scoping (D-10/D-11). It is safe today only because all five `UserTransactionRepository` methods `.Include(t => t.ShopItem)`, and that required-FK Include folds `ShopItemEntity`'s query filter into an inner join that drops cross-group rows. Nothing declares or enforces this invariant. Close the one unguarded call site (`ShopService.ReturnOrSellItemAsync` uses the base `GetByIdAsync`, which has no Include and no scoping) and add a regression test so a future refactor that drops the Include fails loudly.

Purpose: The protection is real but undocumented and fragile — this makes it enforced-by-test and removes the one path that relied on incidental afterward-safety.
Output: A one-line call-site swap in ShopService plus a new UserTransactionRepositoryTests.cs. (The QuestBoardContext.cs comment correction for UserTransaction is handled by plan 49-01, which owns that file.)
</objective>
