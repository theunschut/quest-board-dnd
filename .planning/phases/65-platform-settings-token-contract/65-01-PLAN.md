<objective>
Establish the persistence + Domain-contract foundation for the instance-wide Omphalos integration settings: the singleton `IntegrationSettingEntity`, its DbContext registration, AutoMapper mapping, the plain `IntegrationSetting` domain model, both Domain interfaces (contracts), and the EF Core migration that creates the `IntegrationSettings` table.

This plan defines contracts and schema only — the repository/service implementations and DI wiring land in plan 65-02, and the controller/views in plan 65-03. Interface-first: downstream plans implement and consume the contracts written here.

Purpose: Satisfies SETT-06 (single-row `IntegrationSettingEntity` with columns `OmphalosUrl`, `OmphalosSharedSecret`, `IsEnabled`, instance-wide) and SETT-08 (EF Core migration creates the table).

Output: New entity/model/interface files, modified DbContext + AutoMapper profile, and a generated migration.

Security note (documented tradeoff, not a code change): the shared secret is persisted in plaintext in SQL Server per SETT-06's runtime-editable requirement. This means DB backups now contain a credential usable to forge SSO tokens — a deliberate, accepted posture for a first-party shared-secret bridge (RESEARCH.md Pitfall 2). Flagged here so it is a visible decision at PR review, not an accident.
</objective>
