<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser (SuperAdmin) → controller | Untrusted form input (URL, secret) crosses into the app |
| non-SuperAdmin → /platform/Integrations | Must be rejected before the action body runs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-65-01 | Tampering | CSRF on Index POST + GenerateSecret POST | mitigate | `[ValidateAntiForgeryToken]` on both mutating actions; forms emit the anti-forgery token via the asp-action tag helper. |
| T-65-02 | Elevation of Privilege | Non-SuperAdmin reaching the settings page | mitigate | Controller-level `[Authorize(Policy = "SuperAdminOnly")]` (SETT-07); ASP.NET Core authorization middleware short-circuits before the action runs. Proven by integration tests in plan 65-04. |
| T-65-03 | Information Disclosure | Secret leaked by GET pre-filling the masked field | mitigate | GET never assigns `SharedSecret` from the stored value; renders a `HasSecretConfigured` indicator instead. Only exception: one-render display of a freshly generated secret via `TempData` (never re-read from DB). |
| T-65-05 | Input Validation | Malformed URL | mitigate | `[Url]` + `[StringLength(500)]` on `OmphalosUrl`, server-side `ModelState` validation. |
| T-65-06 | Cryptography | Weak/predictable secret generation | mitigate | `RandomNumberGenerator.GetHexString(64)` (CSPRNG, 256-bit) server-side only — never `Guid`/`System.Random`/client JS (RESEARCH.md Don't Hand-Roll). |
| T-65-SC | Tampering | package installs | accept (N/A) | Zero new packages. |
</threat_model>
