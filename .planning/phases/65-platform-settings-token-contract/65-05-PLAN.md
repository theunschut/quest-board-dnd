    4. Signing sequence (Phase 66), numbered exactly:

       (b) signature = HMACSHA256.HashData(UTF8(secret), payloadBytes);

       (e) ONLY after the signature check passes, JsonSerializer.Deserialize the same decodedPayloadBytes into fields;
       (f) then check expiry against current Unix time with a small clock-skew allowance (plus or minus 30s) and check nonce against the used-token table for replay.

Note: this plan writes a design document; it adds no runtime attack surface. The register records the threats the *contract itself* must foreclose for Phases 66/67.

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-65-C1 | Spoofing / Tampering | Forged token / canonicalization ambiguity between two independently-implemented codebases | mitigate | Contract mandates verify-over-exact-transmitted-bytes (decode then verify then parse), `FixedTimeEquals` for timing-safe comparison, and a single named encoding API — closing both the delimiter-collision and serializer-determinism halves of the canonicalization risk. |

| T-65-C3 | Elevation of Privilege | Replay of a captured token | mitigate | Contract requires an explicit `nonce` (D-01) that Phase 67 tracks in a used-token table on top of the TTL. |
