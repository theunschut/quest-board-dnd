<objective>
Split the current home route. Move the quest board (quest list with signup status) out of `HomeController.Index` into a new `QuestController.Index` action served at `/quests` (D-05), and replace `HomeController.Index` with a simple public landing page (app name + Log In button) that requires no authentication and loads no group-scoped data (D-04). Migrate both quest board view variants from `Views/Home/` to `Views/Quest/`, create new landing-page views in `Views/Home/`, and sweep every in-app reference that pointed to the old quest-board home so it now points to `/quests` (D-07, D-08) — while leaving genuine "return to public landing" links (logout, access-denied, platform-exit) pointing at `/`.

Purpose: Give the app a public front door while keeping the authenticated quest board reachable at a stable URL, and guarantee no in-app navigation accidentally sends a logged-in user to the empty landing page.
Output: 2 controllers modified, 4 views created/replaced, 6 views edited for reference updates.
</objective>
