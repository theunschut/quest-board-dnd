<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Service → Repository | Repository receives visibility inputs (currentUserId, viewerIsDmTier, includeHidden) as explicit params — never reads request/session state itself |
| Contact visibility decision | The three-branch rule lives in an explicit `.Where`, keeping the multi-tenancy query filter (Plan 02) simple and testable |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-57-06 | Information Disclosure | Hidden Contact leaking through the list query | mitigate | Explicit three-branch `.Where` (creator OR revealed OR DM-tier+toggle) per RESEARCH.md Pitfall 2 — not a query filter, tested by Plan 01 facts |
| T-57-07 | Tampering | AutoMapper child-collection replacement corrupting notes on update | mitigate | Dedicated AddNote/UpdateNote/DeleteNote methods on the ContactNotes DbSet; UpdateAsync handles core fields only (RESEARCH.md Pitfall 4) |
| T-57-08 | Repudiation | Note edits losing authorship/timestamp | mitigate | ContactNote carries AuthorUserId + CreatedAt; UpdateNoteAsync stamps UpdatedAt |
| T-57-SC | Tampering | NuGet installs | accept | No packages installed this phase (RESEARCH.md Package Legitimacy Audit: not applicable) |
</threat_model>
