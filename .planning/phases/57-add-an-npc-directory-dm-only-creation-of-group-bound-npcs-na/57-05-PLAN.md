## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| view-rendered controls → controller actions | Buttons/forms are UX affordances only; the controller (Plan 04) enforces authorization — views must not be the security boundary |
| note controls visible to all vs. management controls DM-only | Views gate management on CanManage; note controls are intentionally ungated (D-09) but the controller still requires [Authorize] |

| T-57-15 | Tampering | CSRF via note/toggle/delete forms | mitigate | Antiforgery token on every POST form; controller validates it (Plan 04) |
