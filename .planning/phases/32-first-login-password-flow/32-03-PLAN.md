<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anonymous client → ForgotPassword POST | Untrusted email input; rate-limited; must not leak account existence |
| anonymous client → SetPassword POST | Untrusted password + token from email link; token verified by Identity |
| email link → SetPassword GET | userId + encoded token arrive as query params from an emailed URL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-08 | Information Disclosure | ForgotPassword POST response | mitigate | Identical TempData message + redirect on found and not-found paths (D-11); acceptance criterion enforces single occurrence outside the HasValue branch (email enumeration prevented) |
| T-32-09 | Spoofing / Tampering | SetPassword token | mitigate | Token consumed only via ResetPasswordAsync (ResetPassword purpose); security-stamp binding invalidates the token if the password/email already changed; try/catch on decode returns a generic error for tampered/expired tokens |
| T-32-10 | Elevation of Privilege | ConfirmEmailDirectlyAsync side effect | mitigate | Called only after ResetPasswordAsync returns Succeeded — a valid reset token is a prerequisite, so email-confirmation cannot be forced without proving control of the reset link |
| T-32-11 | Denial of Service | ForgotPassword endpoint | mitigate | [EnableRateLimiting("forgot-password")] applies the 3 req / 15 min per-IP policy from Plan 02 |
| T-32-12 | Input Validation | SetPassword / ForgotPassword models | mitigate | DataAnnotations ([Required], [EmailAddress], [DataType(Password)], [Compare], MinimumLength = 8 matching Identity policy) validated server-side; IdentityResult.Errors surfaced to ModelState |
| T-32-SC | Tampering | npm/pip/cargo installs | mitigate | Zero new packages this plan |
</threat_model>
