Rate-limiter registration (RESEARCH.md verified, Microsoft Learn aspnetcore-10.0):
  using Microsoft.AspNetCore.RateLimiting; using System.Threading.RateLimiting;
  builder.Services.AddRateLimiter(options => {
    options.AddPolicy("forgot-password", httpContext =>
      RateLimitPartition.GetFixedWindowLimiter(
        partitionKey: httpContext.Connection.RemoteIpAddress?.ToString() ?? "unknown",
        factory: _ => new FixedWindowRateLimiterOptions {
          PermitLimit = 3, Window = TimeSpan.FromMinutes(15), QueueLimit = 0, AutoReplenishment = true }));
    options.OnRejected = async (context, ct) => {
      context.HttpContext.Response.StatusCode = StatusCodes.Status429TooManyRequests;
      await context.HttpContext.Response.WriteAsync("Too many requests. Please try again later.", ct); };
  });

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anonymous client → ForgotPassword endpoint | Unauthenticated POST reaches the rate-limited endpoint; IP is the partition key |
| app → email transport (Resend/SMTP) | Rendered email HTML + callback link leave the app across an untrusted channel |
| reverse proxy → app | Client IP seen by the app may be the proxy's IP unless forwarded headers are configured |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-04 | Denial of Service | ForgotPassword rate limiter partition by RemoteIpAddress | mitigate (partial) | Fixed-window 3 req / 15 min per IP registered here. Behind a reverse proxy, RemoteIpAddress may reflect the proxy IP; ForwardedHeaders is confirmed absent. Documented as a manual deploy-environment verification (VALIDATION.md Manual-Only) — acceptable for an internal-tool threat model; not a blocker |
| T-32-05 | Denial of Service | ForgotPassword inbox flooding of a victim | mitigate | 3 req / 15 min per IP throttles single-source floods; distributed floods out of scope for internal tool (RESEARCH.md Security Domain) |
| T-32-06 | Elevation of Privilege | 7-day TokenLifespan widens the reset-token exposure window | accept | Identity tokens embed the security stamp — any password/email change invalidates outstanding tokens automatically. 7 days is an explicit accepted tradeoff (D-13) for a low-traffic internal tool |
| T-32-07 | Information Disclosure | Welcome/ForgotPassword email content | mitigate | ForgotPassword email carries NO user-identifying content (no UserName param on the job/template); Welcome email is only sent to the just-created account's own address |
| T-32-SC | Tampering | npm/pip/cargo installs | mitigate | Zero new packages (RESEARCH.md Package Legitimacy Gate); both RateLimiting + Identity APIs ship in the ASP.NET Core shared framework |
</threat_model>
