## Accomplishments
- `Program.cs` now throws `InvalidOperationException` at Production startup when `Email:FromEmail` or `Email:SmtpServer` is empty, naming only the missing config key(s) — no secret values in the message. Development/Testing unaffected.
- Extracted the Resend HTTP fetch/pagination loop out of `AdminController` into a new `ResendStatsClient.FetchAllRecordsAsync` seam, adding bounded exponential-backoff retry (3 attempts, 1s/2s/4s, `Retry-After` honored) scoped strictly to HTTP 429. Pagination semantics and the per-request Bearer-token injection pattern are preserved exactly.
- Documented the per-request Bearer-token rationale in `ResendStatsClient` and the secret-safe structured-logging pattern in `EmailService.SendAsync`, both via plain (non-ID-tagged) comments — zero behavior change.

## Decisions Made
- Removed `IHttpClientFactory` from `AdminController`'s constructor and the now-dead `AuthenticationHeaderValue`/`JsonSerializer` usings once the HTTP fetch loop moved into `ResendStatsClient` — leaving an unused constructor parameter would be dead code directly caused by this refactor (Rule 1).
- Retry budget fixed at 3 attempts with `TimeSpan.FromSeconds(Math.Pow(2, attempt))` backoff (1s/2s/4s), preferring the `Retry-After` response header when Resend supplies one, and scoped exclusively to `HttpStatusCode.TooManyRequests` — matches the plan's acceptance criteria exactly; no Polly or other new package introduced.
