<objective>
Fix the two actionable Known Bugs and the two documentation/verification Security items for Phase 34.1 that require production-code changes:

1. **Email Settings Not Validated on Startup** — add a Production-only startup guard in `Program.cs` that throws if `Email:FromEmail` or `Email:SmtpServer` is empty, converting today's silent no-op into a fail-fast at boot.
2. **Resend API 429 rate-limit retry** — the pagination loop in `AdminController.GetResendStatsAsync` already works (cursor-based via `afterId`); the genuine gap is that a 429 response just logs and returns an error. Extract the HTTP fetch loop into a testable `ResendStatsClient` seam and add exponential-backoff retry for 429 only. Scope is strictly the stats fetch — do NOT touch `SessionReminderJob`/`DailyReminderJob` per-quest send behavior (that is the deferred digest-batching item).
3. **Resend token HttpClient pattern** — the per-request Bearer injection is already correct; add an explanatory, non-ID-tagged code comment (documentation-only).
4. **Email secret logging** — `EmailService.SendAsync` already logs `ex` as a structured parameter with only `{Subject}` interpolated; verify and add a short non-ID-tagged comment locking the pattern in.

Purpose: Close the Known Bugs and Security Considerations code-change items catalogued in CONCERNS.md as part of closing the v5.0 milestone.
Output: A Production startup email-config guard, a unit-testable `ResendStatsClient` with 429 backoff, and two documentation/verification comments.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| deployment env → app startup | Production config (env-var overrides at /etc/questboard/.env) crosses into the running app; missing email config must fail loudly, not silently drop mail |
| app → Resend API (external HTTP) | Outbound authenticated HTTP; rate limits (429) and the Bearer secret cross this boundary |
| app → structured logs | Exception/log sinks are a lower-trust destination; secrets (SMTP password, API keys, connection strings) must never cross into log messages |

Startup guard (Task 1) message names only config KEYS, never values. |
| T-34.1-03 | Information Disclosure | Resend Bearer token handling (ASVS V13, informational) | accept (already-correct pattern; documented) | Token injected per-request in `ResendStatsClient.FetchAllRecordsAsync`, never a shared HttpClient default header; Task 3 adds a comment documenting the rationale. No behavior change needed. |
| T-34.1-04 | Denial of Service | Resend stats fetch under upstream rate limiting (ASVS V7) | mitigate | Task 2 adds bounded 429 retry-with-backoff (3 attempts, 1/2/4s, Retry-After honored) so transient rate limits degrade gracefully to a surfaced error instead of a hard failure; bounded budget prevents self-inflicted retry storms. |
