# Phase 67 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope note:** This phase mechanically repeats Phase 66's Markdown editor + rendering pattern onto Quest Rewards and Quest Recap, and finishes wiring the email side (Session Reminder, Waitlist Promoted). Every foundational visual decision — design system, spacing scale, `.editor-toolbar` touch-target overrides, `.markdown-content` typography, color palette, third-party registry — was already locked in `66-UI-SPEC.md` and is **global CSS scoped by class name, not by field**, so it applies automatically to Rewards and Recap the moment they're wired to the shared `_MarkdownEditor.cshtml` partial and `Html.Markdown()` helper. No new CSS is required to inherit that contract. This document's job is to (1) confirm that inheritance explicitly so the checker/executor don't re-derive it, and (2) specify the handful of genuinely new copy/markup decisions this phase introduces: the Manage-page Rewards collapsible section (D-01), the `CreateFollowUp.Mobile.cshtml` Rewards field backfill (D-02), and the Recap editor's label/placeholder mapping onto the shared partial. Still not a React/Next.js/Vite project — shadcn gate not applicable, unchanged from Phase 66.

## Spacing Scale

Declared values (must be multiples of 4) — identical to Phase 66, unchanged:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`me-1`), inline padding |
| sm | 8px | Compact element spacing (`me-2`) |
| md | 16px | Default element spacing (`mb-3`) |
| lg | 24px | Section padding (`mb-4`, `.modern-card-body` padding) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Exceptions: none new this phase.**

- The 44px min-width/min-height + 0px-gap `.editor-toolbar button` overrides declared in `66-UI-SPEC.md` live in `markdown-editor.css`, selected by class (`.editor-toolbar button`), not by field name. They apply automatically to every new `markdown-editor-textarea` instance this phase creates (Rewards on 5 write forms, Recap on 2 write forms) with zero additional CSS work.
- Known accepted deviation carried forward unchanged (not re-litigated this phase, per `66-07-SUMMARY.md`): mobile toolbar buttons are 44px tall but only ~30-34px wide due to an EasyMDE-vs-app CSS cascade tie on `min-width`. No overflow occurs; operator explicitly declined to route this to gap closure. This deviation applies identically to the Rewards and Recap toolbars — do not attempt to "fix" it as part of this phase's scope.
- Pre-existing 20px (1.25rem) box padding on `.quest-description-box` (Rewards) and `.recap-display-box` (Recap) — unchanged; only the `white-space: pre-wrap` rule on these two selectors is removed (per `67-CONTEXT.md`'s pre-wrap cleanup decision), the padding stays.
- **Verification note carried forward:** re-confirm the toolbar still fits one row at a true 320px mobile viewport for the Rewards and Recap forms specifically (not just Description) — the arithmetic is identical (7 buttons × 44px = 308px in a 320px budget) but each new form is a distinct DOM context and should be checked, not assumed.

## Typography

No new typography surface — Rewards and Recap render through the exact same `.markdown-content` wrapper Description already uses (`Html.Markdown()` in `HtmlHelperExtensions.cs` wraps every field's output identically, regardless of which field called it). The table below documents the **current as-built state** (post `66-07-SUMMARY.md`'s heading-hierarchy fix, which superseded Phase 66's original flat-1.25rem heading spec) so this phase inherits the correct values, not the stale ones.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (1rem) | 400 regular | 1.6 |
| Label | 16px (1rem) | 700 bold | 1.2 |
| Heading | stepped 24px→14px (h1 1.5rem down to h6 0.875rem) | 700 bold | 1.2 |
| Display | not used | — | — |

**Accepted Deviation — Heading row (6 sizes, exceeds standard 4-size guideline):**

The Heading row's stepped h1–h6 scale (24/22/20/18/16/14px) combined with Body/Label (16px) totals 6 unique font sizes in this document, exceeding the standard 4-font-size guideline. This is a **documented, accepted exception, not a defect to fix by changing CSS**:

- The 6-level heading scale is already-shipped, operator-approved CSS from Phase 66 (`.markdown-content h1`–`h6` in `markdown-content.css`), added at explicit operator request during Phase 66's human-verification checkpoint (see `66-07-SUMMARY.md`).
- It is global, class-scoped CSS applying app-wide to every `Html.Markdown()` render — not introduced by, or specific to, this phase's new Rewards/Recap fields.
- Phase 67 does not modify this CSS. "Fixing" it here would mean silently changing already-shipped, user-requested Description rendering behavior, which is out of Phase 67's scope entirely.
- Accepted as-is by the user, matching the precedent set by Phase 66's own accepted 44px-tall/~30-34px-wide mobile touch-target deviation (see Spacing Scale section above): a known, named, justified departure from a general rule, not a silent violation.

**Notes (inherited, not re-decided):**

- Heading scale is `.markdown-content h1 { 24px }` → `h2 22px` → `h3 20px` → `h4 18px` → `h5 16px` → `h6 14px` (`markdown-content.css`) — applies automatically to any Rewards/Recap heading syntax a user types.
- Heading color `#F4E4BC` with `text-shadow: 1px 1px 3px rgba(0,0,0,0.9)` — matches `.quest-description-box`/`.recap-display-box`'s existing text color exactly, since both fields render inside those dark-overlay boxes.
- Blockquote: `border-left: 3px solid rgba(255, 193, 7, 0.5)`, `padding-left: 12px`, italic — unchanged, inherited automatically.
- Links: unmodified Bootstrap default (`var(--bs-link-color)`, underlined) — unchanged.
- **Editor/Preview text-color specificity fix** (`.CodeMirror.CodeMirror`, `.editor-preview.editor-preview` forcing black text, `markdown-editor.css`) is also a global class selector, not field-scoped — it applies automatically to the Rewards editor (nested in `.modern-card` on `Create.cshtml`/`Edit.cshtml`) and the Recap editor (nested in `.modern-card` on `EditRecap.cshtml`, confirmed by direct read). No new CSS override needed; **do** include a live-browser check of both new editors' text color in this phase's verification checkpoint, since Phase 66's fix was itself found only during human verification, not written into the original plan.
