## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gap (`me-1`, `me-2`) |
| sm | 8px | Compact inline spacing |
| md | 16px | Default form field spacing (`mb-3`), card body side padding |
| lg | 24px | Card body padding (`1.5rem` per `.modern-card-body`) |
| xl | 32px | Section-level separation |
| 2xl | 48px | Page-level empty state padding (`py-5`) |
| 3xl | 64px | Reserved for future layout needs |

Exceptions:
- Touch targets on mobile: minimum 44px height on all tappable elements (Bootstrap `btn-lg` satisfies this on the group picker cards)
- Group picker cards: inner padding `1.5rem` (24px) all sides to create generous tap area

**Source:** `site.css` lines 1063–1068 (`.modern-card-body { padding: 1.5rem }`); Bootstrap `mb-3` = 1rem = 16px at Bootstrap default scale.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (1rem) | 400 (regular) | 1.5 |
| Label | 14px (0.875rem) | 400 (regular) | 1.5 |
| Heading (card) | 20px (1.25rem h2 inside card) | 600 (semibold) | 1.2 |
| Display (group name on picker card) | 20px (1.25rem) | 600 (semibold) | 1.2 |

Rules:
- All headings inside `.modern-card-header` use `<h2 class="mb-0">` — color `#F4E4BC` with dark text-shadow (enforced by `.modern-card-header h2` in `site.css`).
- Body text inside `.modern-card-body` uses `color: #1a1a1a` (enforced by `.modern-card-body`).
- Font Awesome icons always precede text with `me-2` spacing (8px gap).

**Source:** `site.css` lines 1043–1067 + existing view patterns (`Login.cshtml`, `EditUser.cshtml`).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Dark background `#1a1a2e` / `#16213e` (page body) | Full page background (inherited from existing body gradient) |
| Secondary (30%) | `rgba(255,255,255,0.15)` glassmorphism | `.modern-card` backgrounds — all cards on group picker, create user form |
| Accent (10%) | `#ffc107` (Bootstrap warning / amber) | Primary CTA buttons, form focus ring, icon accents |
| Destructive | `#dc3545` (Bootstrap danger / red) | Delete user button, admin-role badge, admin dropdown nav item |

Accent (`#ffc107`) reserved for:
- Primary submit button on Create User form (`btn-warning`)
- Group picker card hover state border and shadow (matches existing `.item-card:hover`)
- Form control focus ring (`border-color: #ffc107; box-shadow: 0 0 0 0.2rem rgba(255,193,7,0.25)` — already in `site.css`)

Destructive (`#dc3545`) reserved for:
- Delete actions only (delete user)
- Admin role badge (`badge bg-danger`)

**SuperAdmin "Go to Platform" button:** Use `btn-secondary` (grey `#6c757d`) — visually distinct from group cards (which use the modern-card interactive pattern) without consuming the accent slot. This button is functional navigation, not a primary CTA.

**Source:** `site.css` lines 1–10 (form focus); `Users.cshtml` role badge colors; `_Layout.cshtml` admin nav `text-danger`; CONTEXT.md D-07.

## View Inventory

Six views are created or modified in this phase. Each has a desktop and mobile variant.

### New Views

#### 1. `GroupPicker/Index.cshtml` — Group Picker Page (desktop)

**Layout:** `_Layout.GroupPicker.cshtml` (new stripped-down layout, modeled on `_Layout.Platform.cshtml` — no main quest board nav, no group context in header since it is not yet set)

**Structure:**
```
<div class="container mt-5">
  <div class="row justify-content-center">
    <div class="col-md-8 col-lg-6">
      [SuperAdmin only: "Go to Platform" btn-secondary btn-lg above cards]
      <div class="card modern-card">
        <div class="card-header modern-card-header">
          <h2><i class="fas fa-layer-group me-2"></i>Select Your Group</h2>
        </div>
        <div class="card-body modern-card-body">
          [card grid — one Bootstrap card per group, clickable via form POST]
        </div>
      </div>
    </div>
  </div>
</div>
```

**Group cards grid:** `display: grid; grid-template-columns: repeat(auto-fill, minmax(200px, 1fr)); gap: 1rem;`

Each group card:
- Bootstrap card with `modern-card` class + `cursor: pointer` hover lift (`transform: translateY(-2px)`)
- Card body: group name as `<h5>` (semibold, `#F4E4BC`), member count as `<small class="text-muted">`
- Icon: `fas fa-users me-2` before member count
- Entire card is wrapped in a `<form method="post" action="/groups/pick/select">` with a hidden `groupId` field and anti-forgery token; `onclick="this.closest('form').submit()"` on the card div
- No separate "Enter" button on each card — the click IS the submit (matches Phase 29 Members view pattern of form-per-row)

**SuperAdmin extras (D-07):**
- A `btn btn-secondary btn-lg` above the cards grid: `<i class="fas fa-cog me-2"></i>Go to Platform` linking to `/platform`
- A `<p class="text-muted mb-3">Or select a group below to enter as a member</p>` separator text

**Zero-groups guard (CONTEXT.md landmines):**
- If authenticated user has no groups and is not SuperAdmin: show `alert-warning` inside the card body: "Your account is not assigned to any group. Please contact your administrator."
- No group cards rendered in this state

#### 5. `Views/Shared/_Layout.cshtml` — Nav Group Switch Item

**Change:** Inside the authenticated user dropdown (after Profile `<li>`, before Logout `<li>`), add:


The active group name is shown as a static `<span>` in the nav (separate from the switch link), OR the switch link text IS the group name depending on D-16 resolution. If `ViewBag.ActiveGroupName` is null (SuperAdmin context, no active group), render: `<i class="fas fa-arrows-rotate me-2"></i>Switch Group`.

## Interaction Contract

### Group Picker — Card Click Behavior (D-06, CONTEXT.md specifics)

- Each group card is wrapped in a `<form>` with `method="post"` action targeting `GroupPicker/SelectGroup`
- Card div has `onclick="this.closest('form').submit()"` and `style="cursor:pointer"`
- Anti-forgery token included in every form (`@Html.AntiForgeryToken()`)
- No JavaScript libraries required beyond jQuery (already loaded in layout)

### Group Picker — Auto-Redirect for Single-Group Users (UX-01)

- `GroupPickerController.Index` GET: if `User.Groups.Count == 1` → immediately set session + issue HTTP 302 to `returnUrl` or `/`
- No picker page rendered — user never sees the picker
- This is controller logic, not a UI state, but the executor must not add a "loading" indicator — the redirect is synchronous

### Create User — GroupRole Dropdown

- Default selection: Player (first option)
- Options rendered via `asp-items="Html.GetEnumSelectList<GroupRole>()"` — uses enum display names
- Display names must match: "Player", "Dungeon Master", "Admin" (not raw enum names)

### Promote/Demote Existing Users (MGMT-08)

- Already satisfied by existing `AdminController` promote/demote actions updated in Phase 29
- No new UI required; only the `?? 1` fallback removal (D-17) is needed
- The existing Users table with promote/demote buttons is the complete UI for MGMT-08

### ReturnUrl Threading

- Login POST redirects to `/groups/pick?returnUrl={encoded}` — `returnUrl` is URL-encoded
- Picker `SelectGroup` POST redirects to `returnUrl` (validated as local URL) or `/` if absent
- No UI element needed; this is a controller concern, but executor must test with a non-trivial `returnUrl`

## Accessibility

- All interactive cards must be keyboard-navigable: card forms must be focusable; submit on Enter key within form (default browser behavior satisfies this)
- Color contrast: `#F4E4BC` text on dark glassmorphism cards meets WCAG AA (existing pattern, validated in prior phases)
- `<i>` icons are decorative — `aria-hidden="true"` not required (existing project convention omits it; maintain consistency)
- Error messages use `role="alert"` via Bootstrap `alert` classes (existing pattern)
