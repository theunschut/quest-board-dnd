<objective>
Wire login into the group-picker flow and remove public self-registration.

Purpose: D-01 mandates every successful login route through `/groups/pick` so the group-context lifecycle (plan 30-01) runs for all users. REG-01 removes the public registration path — accounts are now created only by admins (plan 30-03). Removing the Register actions and views, plus the "Create Account" links on the login pages, closes the public sign-up surface.

Output: `AccountController.Login` POST redirects to `GroupPickerController.Index`; `AccountController.Register` GET/POST removed; `Register.cshtml` + `Register.Mobile.cshtml` deleted; "Create Account" links removed from `Login.cshtml` and `Login.Mobile.cshtml`.

Implements D-01, D-09, D-19 — covers REG-01 and is a prerequisite for UX-01/UX-02/UX-03 (login is the entry point to the picker).
</objective>
