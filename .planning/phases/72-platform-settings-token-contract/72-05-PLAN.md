<objective>
Build the Group Admin-only per-group Omphalos override page, a sibling of the instance-wide page but living in the group-scoped Admin surface. It writes/reads `GroupId == <active group>` rows through the same service, communicates the cascade state (Override Active / Inherited / Not Configured) to a Group Admin who may not know an instance-wide default exists, and supports clearing the override to fall back to the default.

Purpose: A group can point at its own Omphalos deployment/secret independently of the platform default (or inherit it). The authorization this needs already exists: `[Authorize(Policy = "AdminOnly")]` is backed by `AdminHandler`, which enforces exactly `GroupRole.Admin` for the active group (with SuperAdmin bypass and a null-active-group guard) and excludes DungeonMaster — so no new authorization plumbing is required.
Output: A working, Admin-gated, mobile-paired group-override page reachable from the Admin navbar dropdown.

Implements: SETT-09 (group Admin configures a group-specific override from the group Admin area; applies only to that group), SETT-10 (DungeonMaster cannot configure it — Group Admin only). Realizes D-09 (two separate pages; group override is Group Admin only, not DM), and reuses D-03 (Generate Secret, scoped to the group).
</objective>
