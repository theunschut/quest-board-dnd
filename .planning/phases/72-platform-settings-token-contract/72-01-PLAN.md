<objective>
Lay the persistence foundation for the Omphalos settings feature using the generic key-value design locked in CONTEXT.md D-07/D-08/D-10 (superseding the rejected fixed-column `IntegrationSettingEntity` singleton). This plan creates the Domain contracts (model, DTO, key constants, repository + service interfaces) and the EF Core layer (entity, AutoMapper map, DbContext registration with provider-safe filtered unique indexes, and the migration).

Purpose: Every downstream surface — the SuperAdmin instance-wide page (72-04), the Group Admin override page (72-05), and Phase 73's token generator — resolves settings through the interfaces defined here. Getting the entity shape, the nullable-GroupId cascade semantics, and the uniqueness guarantees right now prevents rework across all of them.
Output: Compilable Domain contracts + EF entity + `AddPlatformSettings` migration. No behavior wiring yet (repository/service implementations land in 72-03).

Implements: SETT-06 (generic key-value `PlatformSettingEntity` with `Key`/`Value`/nullable `GroupId`), SETT-08 (EF Core migration creates the settings table). Realizes D-07 (key-value not fixed-column), D-08 (GroupId null = instance default, cascade design), D-10 (only `(Key, GroupId)` unique; duplicate Values allowed).
</objective>
