## What This Is

A D&D campaign management web application for a group of players and Dungeon Masters. It handles quest creation and scheduling, player signup with date voting, a character/guild system, a shop with gold economy, and email notifications (HTML-templated, Hangfire-dispatched, Resend-relayed). Groups choose a board type at creation ‚Äî a One-Shot board (date voting + finalization, the original flow) or a Campaign board for ongoing games with a fixed party (no date voting/signup, simple Close/Reopen lifecycle, no scheduling emails). Built with ASP.NET Core 10 MVC, SQL Server, and Docker ‚Äî deployed as a single container to a self-hosted Linux environment.

## Core Value

The quest board must reliably let DMs post quests and players sign up ‚Äî everything else enhances that loop.
