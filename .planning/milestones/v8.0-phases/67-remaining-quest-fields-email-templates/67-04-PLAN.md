<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| stored Quest Description → rendered email HTML (server-rendered, delivered to external mail clients) | User-authored Markdown becomes HTML injected via `MarkupString` into an outbound email. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-67-02 | Tampering / Information Disclosure (XSS + external image tracking) | `MarkupString` of `RenderToHtml(QuestDescription, Email)` in SessionReminder/WaitlistPromoted | mitigate | The `Email` render target sanitizes via HtmlSanitizer AND strips `<img>` (blocks tracking-pixel / external-image loading in mail clients) — pinned by the image-strip assertion in each new test. `MarkupString` wraps only sanitizer output, never raw `@QuestDescription`. Same pipeline QuestFinalized already ships. |
| T-67-02-B | Denial of Service / layout break (structure dropped) | `<p>` illegally wrapping block-level Markdown output | mitigate | Use `<div>`, not `<p>` (CR-01). The multi-block test asserts the styled wrapper is not self-emptied, preventing silent structure loss. |
| T-67-SC | Tampering (supply chain) | dependencies | accept | No new package installs; Markdig/HtmlSanitizer already shipped in Phase 65. |
</threat_model>
