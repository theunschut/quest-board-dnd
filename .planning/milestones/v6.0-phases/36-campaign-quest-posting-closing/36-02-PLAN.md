<objective>
Implement the campaign Close/Reopen lifecycle at the Domain + Repository layers and wire `IsClosed` into the three quest-visibility filters, driven by unit tests written first.

Purpose: This is the business-logic core of CQUEST-03/04/05/06. `CloseQuestAsync`/`ReopenQuestAsync` mirror the existing `OpenQuestAsync` shape exactly, but must NEVER call `IQuestEmailDispatcher` (that omission is the structural mechanism for CQUEST-06 — a campaign quest never sets `FinalizedDate`, so `DailyReminderJob`/`QuestFinalizedEmailJob` remain blind to it). The board filters gain `!IsClosed` so closing hides a quest immediately; `GetCompletedQuestsAsync` gains an OR-branch so a closed quest shows in the Quest Log with no next-day wait while one-shot finalization keeps its next-day wait unchanged.
Output: Two new service+repository methods (with interface signatures), three edited filter predicates, and a suite of unit tests covering close/reopen state transitions, board exclusion, Quest Log inclusion, and zero-email assertions.
</objective>
