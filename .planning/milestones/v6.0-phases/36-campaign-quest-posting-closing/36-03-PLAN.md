## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → QuestController POST | Untrusted form input (quest id, ProposedDates, ChallengeRating, TotalPlayerCount) crosses here |
| Browser → QuestController Create | Client could tamper with fields the campaign UI hides; BoardType must never be trusted from the post |
| Client identity → quest ownership | A DM could attempt to close another DM's quest (elevation) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-36-05 | Elevation of Privilege | `Close`/`Reopen` actions | mitigate | Reuse `IsQuestOwner(currentUser, quest.DungeonMaster) \|\| role == GroupRole.Admin` verbatim (Phase 34.3-fixed `User.Id`-based check); `Forbid()` otherwise. Verified by QuestCloseTests non-owner-non-admin → 403 test. |
| T-36-06 | Tampering / Spoofing | `Close`/`Reopen` POST endpoints | mitigate | `[ValidateAntiForgeryToken]` on both actions (matches every mutating action in this controller). Verified by QuestCloseTests missing-token → rejected test. |
| T-36-07 | Tampering | Campaign Create field-stripping bypass | mitigate | `BoardType` resolved server-side via `GetActiveBoardTypeAsync` (never a bindable ViewModel property); campaign Create silently overrides `ProposedDates`/`ChallengeRating`/`TotalPlayerCount`/`DungeonMasterSession` to fixed defaults regardless of posted values. Matches Phase 35 D-06 tamper philosophy. |

No new code; enforced by mirroring `Open`/`Finalize` exactly. |
| T-36-SC | Tampering | npm/pip/cargo installs | accept | No package installs — pure C# on already-referenced packages. No `[ASSUMED]`/`[SUS]` packages. |
