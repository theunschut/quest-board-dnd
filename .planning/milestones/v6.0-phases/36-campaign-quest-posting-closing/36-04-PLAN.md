<objective>
Render the campaign quest experience across the board, Manage, Details, and Create views (desktop + mobile): swap the wax seal to Open/Closed for campaign, drop the CR badge and signup/date-voting sections that don't apply, add Close/Reopen buttons on Manage, and hide the campaign-irrelevant Create fields — all gated on `ViewBag.BoardType`, with one-shot rendering untouched.

Purpose: This is the visible surface of CQUEST-01/02/03/04. Per 36-UI-SPEC.md this phase adds ZERO new views and ZERO new CSS — every change is a conditional wrap or a text/class swap on existing markup. The wax seal is reused verbatim (D-06 resolution): same two CSS classes (`finalized-seal`/`open-seal`), same seal-image pool, only the driving condition and alt text change for campaign. Removals are full `@if` removals (not `display:none`) so the flexbox layout reflows and the description grows into the freed space (D-04/D-05). Close uses `btn-secondary` (quiet, no confirm), Reopen uses `btn-warning` (no confirm — no destructive side effect, unlike one-shot Open).
Output: Eight edited Razor views (four desktop + four mobile) and a human-verify checkpoint confirming the campaign board/manage/create/details experience.
</objective>
