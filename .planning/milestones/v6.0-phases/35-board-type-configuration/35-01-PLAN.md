| Boundary | Description |
|----------|-------------|
| SuperAdmin browser -> Platform GroupController | Form POST data crosses from an authenticated SuperAdmin into the persistence layer. Only SuperAdmins reach these actions (`[Authorize(Policy = "SuperAdminOnly")]` at class level). |
| Domain model -> EF entity (AutoMapper) | Enum-to-int conversion boundary; a mis-mapping could silently store the wrong ordinal. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-01 | Tampering | `GroupController.Edit` POST (mass-assignment / over-posting of BoardType) | mitigate | Enforced by ViewModel shape in Plan 02 (BoardType is never read from the posted model into the entity). This plan lays the groundwork by scaffolding `EditGroup_PostingChangedBoardType_ShouldBeSilentlyIgnored`, which fails until the mitigation is proven. | T-35-02 | Tampering | AutoMapper `Group<->GroupEntity` int<->enum cast | accept | Explicit `ForMember` casts (not bare `ReverseMap()`) make the ordinal conversion deterministic and match the codebase-wide convention for all 10 other enums. | T-35-03 | Denial of Service | EF migration on existing `Groups` rows | accept | `AddColumn<int>(..., defaultValue: 0)` is an additive, non-destructive column add with a server-side default — no data rewrite, no lock beyond the normal DDL.
