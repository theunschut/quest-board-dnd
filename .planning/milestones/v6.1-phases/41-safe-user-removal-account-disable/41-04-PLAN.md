<objective>
Update `AccountController.Login`'s `IsLockedOut` branch (SAFE-04) so a disabled account (`LockoutEnd == DateTimeOffset.MaxValue`) shows an accurate "This account has been disabled" message, while an ordinary 5-failed-attempt temporary lockout keeps the existing "Try again in 15 minutes" copy.

Purpose: Today, any `result.IsLockedOut` — whether from a real 15-minute failed-attempt lockout or from this milestone's new disable feature — shows the same "Account locked due to too many failed attempts. Try again in 15 minutes." message, which is misleading for a deliberately-disabled account. This plan composes the existing `IIdentityService.GetIdByEmailAsync(email)` with Plan 02's `GetLockoutEndAsync(userId)` to distinguish the two cases via an exact `== DateTimeOffset.MaxValue` sentinel comparison (D-13), because `SignInResult.LockedOut` carries no `LockoutEnd` payload (RESEARCH Pitfall 4).

Output: An updated `IsLockedOut` branch and integration tests covering both message paths.
</objective>
