<objective>
Add session-recovery middleware so an authenticated user whose group session has expired (no active group) is redirected to the group picker instead of hitting a broken, group-scoped page (D-09, D-10, D-11). The middleware resolves the active group through `IActiveGroupContext` (which in production reads `SessionKeys.ActiveGroupId` from `ISession` via `ActiveGroupContextService`, and in tests is the settable `MutableGroupContext`), exempts the picker/auth/platform/error paths, and short-circuits for SuperAdmin (who legitimately has a null active group). Also add `[Route("groups/pick")]` to `GroupPickerController.Index` so the CONTEXT.md vanity URL `/groups/pick` is a real route that both the redirect target and the exempt-path check can rely on.

Purpose: Turn the "expired group session" failure mode into a seamless recovery (single-group users auto-re-enter via the picker's auto-pick; multi-group users re-pick) without exposing broken UI or causing redirect loops.
Output: One new middleware class, one Program.cs registration, one route attribute on GroupPickerController.
</objective>

  <action>
    Create `QuestBoard.Service/Middleware/GroupSessionMiddleware.cs` in namespace `QuestBoard.Service.Middleware`. Define `public class GroupSessionMiddleware(RequestDelegate next)` with `public async Task InvokeAsync(HttpContext context)`. The guard logic, evaluated in this order:
    1. If the user is NOT authenticated (`context.User.Identity?.IsAuthenticated != true`) → call `await next(context)` and return. (ASP.NET Identity's `[Authorize]` handles anonymous redirects; the middleware only handles the authenticated-but-no-group case.)
    2. If the user IS in role "SuperAdmin" (`context.User.IsInRole("SuperAdmin")`) → `await next(context)` and return. (SuperAdmin has a null active group by design — redirecting would loop. This MUST be checked before the group check.)
    3. If the request path is exempt → `await next(context)` and return. Exempt when `context.Request.Path.StartsWithSegments` matches any of: `/groups/pick`, `/GroupPicker` (the conventional route for the same controller — both must be exempt so neither the vanity nor the default URL loops), `/Account`, `/platform`, `/Error`. Use case-insensitive segment matching (StartsWithSegments is case-insensitive by default).
    4. Resolve `var groupContext = context.RequestServices.GetRequiredService<IActiveGroupContext>();` and if `groupContext.ActiveGroupId == null` → `context.Response.Redirect("/groups/pick"); return;` (do NOT call next — short-circuit the pipeline). The redirect target is the hardcoded literal `/groups/pick`, never a user-supplied URL (open-redirect mitigation).
    5. Otherwise → `await next(context)`.
    Add the necessary usings: `Microsoft.AspNetCore.Http`, `Microsoft.Extensions.DependencyInjection` (for GetRequiredService), and the namespace that declares `IActiveGroupContext` (QuestBoard.Domain.Interfaces). Do NOT read `context.Session` directly — go through `IActiveGroupContext` so the middleware is testable via MutableGroupContext and stays consistent with the production session-read in ActiveGroupContextService.
  </action>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated client (expired group session) → group-scoped page | A logged-in user with no active group requests a page that needs one |
| middleware → redirect target | The middleware issues an HTTP redirect; the target must not be attacker-controllable |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-06 | Spoofing (open redirect) | GroupSessionMiddleware redirect | mitigate | Redirect target is the hardcoded literal `/groups/pick`; no user-supplied URL is ever passed to `Response.Redirect` |
| T-31-07 | Elevation of Privilege / DoS (redirect loop) | SuperAdmin with null ActiveGroupId | mitigate | Explicit `IsInRole("SuperAdmin")` early-out before the group check; `/groups/pick` and `/GroupPicker` are exempt so the destination never loops |
| T-31-08 | Information Disclosure | Group-scoped pages reached without an active group | mitigate | Middleware redirects to the picker before the controller runs, preventing rendering of a broken/cross-group state |
| T-31-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase — RESEARCH.md confirms zero external dependencies added |
</threat_model>
