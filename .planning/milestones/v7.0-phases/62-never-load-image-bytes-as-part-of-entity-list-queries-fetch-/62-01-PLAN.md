<objective>
Remove eager image-byte loading from the six Character/Contact/DungeonMaster read methods and replace each with a projected boolean (`HasProfilePicture` / `HasContactImage`) computed by a scalar query that never selects the image byte columns. Add the new booleans as additive properties on the three Domain models (keeping the existing byte[] properties untouched), and declare them `.Ignore()` on the AutoMapper Entity→Domain maps so the repository sets them explicitly post-map.

Purpose: This is the backend read-projection foundation for Phase 62. Every downstream layer (services, viewmodels, controllers, views) depends on the Domain models carrying these booleans and on the read methods no longer transferring image bytes.

Output: Three Domain models with a new bool each; six repository read methods with `.Include(...ProfileImage)` removed and a boolean projection added; three `.Ignore()` AutoMapper entries; extended repository unit tests proving the boolean is true-with-image / false-without.
</objective>
