<objective>
Establish the data-model foundation and the two repository/ordering primitives the rest of Phase 44 composes: add the `LastVoteChangeTime` column, generalize the buggy `ChangeVoteToYesAndSelectAsync` into a `ChangeVoteAsync(vote)` method that fixes the VoteType-int bug and stamps the ordering timestamp, add a `GetTopWaitlistedCandidateAsync` query, and centralize waitlist ordering into a single shared extension so desktop and mobile can never drift.

Purpose: VOTE-01 (never reject on capacity), VOTE-02 (vote-priority-then-time ordering), and VOTE-03 (reset ordering timestamp on every vote change) all bottom out in these primitives. Getting them correct and tested first lets Plan 02 (orchestration) and Plan 03 (UI) stay thin.
Output: A migrated `PlayerSignups.LastVoteChangeTime` column, a corrected + generalized repository vote method, a top-candidate query, a `WaitlistOrdering` extension, and the Wave 0 test scaffolding covering VOTE-01/02/03 plus the enum-cast regression.
</objective>
