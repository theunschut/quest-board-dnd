<objective>
Remove the DM/Admin-only gate on editing a completed quest's Session Recap so any authenticated member of the quest's group can edit it, while preserving the DM/Admin-only gate on the unrelated "Manage Quest" link (which currently shares the same ViewBag flag). The recap-edit button and page open to all viewers; the Manage Quest link stays locked to the quest's DM or an Admin.

Purpose: Implements the user's request ("not all DMs want to write recaps, some players do — leave recap editing open to all") without silently exposing the full Manage Quest surface to every player (a permission-escalation bug the current shared-flag design would introduce).
Output: Updated QuestLogController (auth removed from EditRecap, new CanManageQuest flag), both Details views with a two-flag split, and updated integration tests that flip the three Player-denial assertions to Player-allowed and add Manage-link gating coverage.
</objective>
