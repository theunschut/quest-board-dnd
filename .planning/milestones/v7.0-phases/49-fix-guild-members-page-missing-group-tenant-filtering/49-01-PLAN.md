<objective>
Give `CharacterEntity` a real `GroupId` column, an EF Core global query filter (deliberately without the `ActiveGroupId == null ||` escape hatch that Quest/ShopItem use), and a creation-time group stamp — closing the Guild Members cross-group leak (D-01 through D-04) at the data-access layer so every current and future Character query is automatically tenant-scoped. Also rewrite the profile-picture query to route through the filtered `CharacterEntity` root, and settle ALL `QuestBoardContext.cs` transitive-filter comment corrections (Character, UserTransaction, PlayerSignup) in one place so downstream plans 03/04 never touch this file.

Purpose: Character list/detail/picture endpoints currently return every character in the database regardless of group. This plan makes the fix durable (schema + automatic filter) instead of a manual join a future developer could forget.
Output: A migration, an entity column + navigation, a query filter, a rewritten repository method, a controller stamp, a new unit-test file, and a test-helper fix.
</objective>
