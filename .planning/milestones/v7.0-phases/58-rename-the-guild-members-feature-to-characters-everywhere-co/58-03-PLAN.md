<objective>
The atomic core of the rename. Rename `GuildMembersController` → `CharactersController` AND move `Views/GuildMembers/` → `Views/Characters/` in the SAME task/commit, then update the guild-* class references and user-facing copy inside the four content-bearing view files. These three sub-changes are interdependent by MVC's default view-location convention (`Views/{ControllerName}/{Action}.cshtml`): a renamed controller with the old view folder → every action 404s "view not found"; the old controller with a renamed view folder → same. They must never be split across tasks that could land out of order (RESEARCH.md Pitfall 1).

Depends on Plan 02 (Wave 1): the mobile Index view's `<link href>` and its class references must point at the already-renamed `characters.mobile.css` and its `character-*` classes.

Purpose: Move the character-roster route from `/GuildMembers/*` to `/Characters/*` and align every view with the Character naming, with zero behavior change.
Output: `CharactersController` serving `/Characters/*`; `Views/Characters/` with all 8 views; guild-free Index/Details copy and classes.
</objective>
