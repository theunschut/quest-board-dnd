| Details-page block heading (desktop, `Quest/Details.cshtml`) | `Rewards` with `<i class="fas fa-coins text-warning me-2"></i>` icon prefix — no heading currently precedes Description on this specific view (unlike QuestLog/Details), so introduce a matching `<h5>` heading immediately above the new box for visual parity with the QuestLog page's "Original Quest Description" heading pattern |
| Details-page block heading (QuestLog `Details.cshtml`) | `<h5 class="text-white"><i class="fas fa-coins text-warning me-2"></i>Rewards</h5>` — exact class parity with the existing `<h5 class="text-white"><i class="fas fa-scroll me-2"></i>Original Quest Description</h5>` heading it sits beside |
| Mobile block heading | `<i class="fas fa-coins me-1"></i>Rewards` sized/classed like other mobile section headings (`.quest-section-heading` treatment where a block is used, or inline within `.quest-header-card-mobile` sitting under Description per D-06's box treatment) |
| Empty state | No copy — block is hidden entirely when Rewards is null/empty/whitespace (D-07). No "No rewards" placeholder text anywhere. |
| Error state | Not applicable — no validation errors possible on an unbounded, optional, unattributed string field; no `[Required]`/`[StringLength]` means no new validation message is ever shown. `<span asp-validation-for="Rewards" class="text-danger"></span>` should still be present for consistency with sibling fields, but will never render text under normal operation. |
| Destructive confirmation | Not applicable — this phase adds no delete/revoke/destructive action |
| Follow-Up form field | Same label (`Rewards`) and placeholder as Create/Edit, but rendered **blank** (no pre-fill) per D-04 — no special "starts empty" copy needed, this is implicit from the empty textarea |

## Interaction & Layout Notes (phase-specific, supplements the contract above)

These notes translate the contract above into per-file guidance, cross-referencing CONTEXT.md's canonical_refs so the executor doesn't have to re-derive placement:

1. **Create.cshtml / Create.Mobile.cshtml / Edit.cshtml / Edit.Mobile.cshtml / CreateFollowUp.cshtml** — insert a `<div class="mb-3">` block identical in structure to the Description field block, immediately after it and before the `@if (boardType != BoardType.Campaign)` conditional (Create/Edit) or before Challenge Rating (CreateFollowUp, which has no board-type conditional):
   ```html
   <div class="mb-3">
       <label asp-for="Rewards" class="form-label">Rewards</label>
       <textarea asp-for="Rewards" class="form-control" rows="4" placeholder="Describe the loot, gold, or other rewards for this quest..."></textarea>
       <span asp-validation-for="Rewards" class="text-danger"></span>
   </div>
   ```
   For Edit views, `asp-for="Quest.Rewards"` (matches the `Quest.Description` pattern already used there). No `<span class="text-danger">*</span>` required marker — field is optional.

2. **Quest/Details.cshtml** — insert immediately after line 31's `<p class="lead" ...>@Model.Quest?.Description</p>`, gated by D-07's hide-when-empty rule:
   ```html
   @if (!string.IsNullOrWhiteSpace(Model.Quest?.Rewards))
   {
       <div class="mb-4">
           <h5><i class="fas fa-coins text-warning me-2"></i>Rewards</h5>
           <div class="quest-description-box" style="white-space: pre-wrap;">@Model.Quest.Rewards</div>
       </div>
   }
   ```
   Reuse `.quest-description-box` directly (identical CSS to `.recap-display-box` per `quests.css:813-834` — no new class needed, per CONTEXT.md's discretion note). Add `white-space: pre-wrap;` inline (or fold it into a shared class) since `.quest-description-box` itself doesn't declare it but `.recap-display-box` does — match `.recap-display-box`'s behavior for consistency with multi-line rewards text.

3. **Quest/Details.Mobile.cshtml** — mirror the existing Description conditional block exactly (lines 97-103), placed immediately after it inside `.quest-header-card-mobile`:
   ```html
   @if (!string.IsNullOrWhiteSpace(Model.Quest?.Rewards))
   {
       <hr class="quest-card-divider mt-2 mb-2" />
       <div class="quest-description-mobile">
           <i class="fas fa-coins me-1"></i>@Model.Quest.Rewards
       </div>
   }
   ```
   Same `.quest-description-mobile` class as Description (no new CSS class) — the icon prefix is the only visual differentiator between the Description and Rewards mobile blocks, since both otherwise share identical styling.

4. **QuestLog/Details.cshtml** — insert a new `<div class="mb-4">` block after the existing "Original Quest Description" block (lines 44-50) and before the `<hr />`/Adventurers section, per CONTEXT.md's discretion default:
   ```html
   @if (!string.IsNullOrWhiteSpace(Model.Quest.Rewards))
   {
       <hr />
       <div class="mb-4">
           <h5 class="text-white"><i class="fas fa-coins text-warning me-2"></i>Rewards</h5>
           <div class="quest-description-box">
               @Model.Quest.Rewards
           </div>
       </div>
   }
   ```
   Reuse `.quest-description-box` directly (per CONTEXT.md's discretion — no new same-shaped class needed). Note this box already carries `white-space` behavior appropriately for the Recap box; confirm pre-wrap is inherited/adequate for Rewards' potential multi-line text (mirror whichever of `.quest-description-box`/`.recap-display-box` the executor picks, both are visually identical per D-06).

5. **No changes to `_QuestCard.cshtml` or `Quest/Index.cshtml`** per D-05 — do not add any Rewards indicator, icon, or snippet to the quest board list card.

6. **Icon choice**: `fa-coins` is locked by D-06 for every Rewards heading/label across desktop and mobile — do not substitute `fa-gem`, `fa-treasure-chest`, or any other icon.
