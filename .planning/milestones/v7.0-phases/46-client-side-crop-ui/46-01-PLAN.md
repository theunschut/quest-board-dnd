<objective>
Widen the three Domain image-update methods so a caller can thread genuinely new cropped image bytes through to the repository, and add the matching `CroppedPictureFile` binding property to the three ViewModels. Today `CharacterService.UpdateAsync`, `ContactService.UpdateAsync`, and `DungeonMasterProfileService.UpsertProfileAsync` all hardcode the cropped column to `null` on a new-original upload — there is no parameter through which a real crop can flow. This is the backend gap that must exist before the crop UI has anywhere to submit its output.

Purpose: Unblock the entire phase — every later plan (controller wiring, crop modal) submits into these widened signatures.
Output: 3 widened Domain service methods (additive overloads / new parameter), 3 ViewModels with a `CroppedPictureFile` property, and extended service unit tests covering the three crop-resolution branches (Wave 0 test coverage).
</objective>
