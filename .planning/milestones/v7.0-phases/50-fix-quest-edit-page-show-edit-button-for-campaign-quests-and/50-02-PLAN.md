<objective>
Add the missing Edit and Delete affordances to the Campaign board-type action row on both the desktop and mobile Manage pages, achieving parity with the OneShot action row. This closes the primary bug from the user's report (Campaign quests have no Edit button on Manage) plus the closely-related Delete gap surfaced during discussion (D-03).

Purpose: Campaign quests are today only editable/deletable via indirect routes (the Delete button on the Quest Index card); the Manage page — the DM's main per-quest control surface — offers only Close/Reopen. This makes the Edit and Delete actions reachable from the natural place, using the exact buttons OneShot quests already have.

Output: Edited `Manage.cshtml` and `Manage.Mobile.cshtml`. No new controller action, no new JS, no new CSS — every added element is a verbatim reuse of markup already present in the same file for OneShot quests.
</objective>
