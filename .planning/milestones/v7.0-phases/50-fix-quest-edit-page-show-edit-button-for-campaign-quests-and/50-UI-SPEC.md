# Phase 50 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope note:** This is a narrow UI-parity bug fix, not new design. ## Interaction Contract

### Manage page — Campaign action row (D-01, D-02, D-03)

Desktop (`Manage.cshtml`, inside the existing `@if (boardType == BoardType.Campaign)` block at lines 524-547):

Row order, left group: **Edit Quest → Close/Reopen Quest → Delete Quest**. Right side (unchanged): `Refresh Data`.

```html
<a href="@Url.Action("Edit", "Quest", new { id = Model.Id })" class="btn btn-primary">
    <i class="fas fa-edit me-1"></i>Edit Quest
</a>
<!-- existing Close/Reopen form(s) unchanged, add ms-2 for spacing consistency with the row -->
<a href="#" class="btn btn-danger ms-2" onclick="deleteQuest(@Model.Id)">
    <i class="fas fa-trash me-1"></i>Delete
</a>
```

Mobile (`Manage.Mobile.cshtml`, inside the existing `@if (boardType == BoardType.Campaign)` block at lines 372-396), same order, using the mobile button shape (`flex-fill`/`w-100` per existing convention in that block):

```html
<a href="@Url.Action("Edit", "Quest", new { id = Model.Id })" class="btn btn-primary flex-fill">
    <i class="fas fa-edit me-1"></i>Edit Quest
</a>
<!-- existing Close/Reopen form(s) unchanged -->
<a href="#" class="btn btn-danger w-100" onclick="deleteQuest(@Model.Id)">
    <i class="fas fa-trash me-1"></i>Delete Quest
</a>
```

Note: D-02 places Edit Quest **before** Close/Reopen; D-03 places Delete Quest **last**, after Close/Reopen — mirroring the OneShot row's Finalize → Edit → Delete convention (content actions before state-transition actions, destructive action last).

### Edit page — field-hiding conditional (D-04, D-05)

`Edit.cshtml` and `Edit.Mobile.cshtml`: wrap the same 4 field blocks Create already wraps, in the identical condition:

```csharp
@if (boardType != BoardType.Campaign)
{
    <!-- Challenge Rating field (Edit.cshtml:34-39) -->
    <!-- Total Player Count field (Edit.cshtml:42-46) -->
    <!-- DungeonMasterSession checkbox (Edit.cshtml:48-56) -->
    <!-- Proposed Dates block, including HasExistingSignups warning banner (Edit.cshtml:58-92) -->
}
```

Both views need `var boardType = (BoardType)ViewBag.BoardType;` declared at the top (mirroring `Create.cshtml:7`), which requires `QuestController.Edit` GET to set `ViewBag.BoardType = await GetActiveBoardTypeAsync(token);` (D-05) — no view can branch on `boardType` without this controller change landing first.

No read-only/disabled variant of these fields is shown for Campaign quests — full hide, matching Create exactly (D-04).

### Sidebar tips (D-06 — explicitly unchanged)

`Edit.cshtml`'s "Quest Editing Tips" sidebar is NOT touched by this phase. It will continue to reference dates/votes regardless of board type, matching `Create.cshtml`'s identical pre-existing gap. Do not add a conditional here — this is an intentional non-fix, not an oversight.
